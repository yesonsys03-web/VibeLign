## VibeLign v2.5.14 - Windows 미리보기 HTTP 로딩 수정

### 핵심 수정

- Windows `실행해보기` 미리보기 창에서 정적 `index.html` 앱이 빈 화면으로 멈추던 문제를 다시 수정했다.
- v2.5.13의 앱 내부 커스텀 프로토콜 경로를 제거하고, 브라우저와 같은 `http://localhost:<port>/index.html` 방식으로 미리보기를 열도록 변경했다.
- 정적 미리보기 서버가 공백 등 인코딩된 asset 경로를 해석하도록 보강했다.
- 외부 URL 이동 차단은 유지하되, 정적 앱 실행에 필요한 `about:blank`, `data:`, `blob:` 내부 WebView 이동은 계속 허용한다.

### 검증

- `cargo test --manifest-path vibelign-gui/src-tauri/Cargo.toml commands::run_preview -- --nocapture`
- `cargo check --manifest-path vibelign-gui/src-tauri/Cargo.toml`
- `npm --prefix vibelign-gui run build`

### 테스트 안내

- Windows에서 `index.html`만 있는 프로젝트를 열고 `실행해보기`를 눌러 미리보기 창에 실제 UI가 표시되고 조작되는지 확인한다.
- 같은 `index.html`을 브라우저에서 직접 열었을 때와 VibeLign 미리보기 창의 동작이 같은지 비교한다.
- 미리보기 중 `중지` 버튼을 눌렀을 때 실행 상태가 즉시 해제되는지 확인한다.
