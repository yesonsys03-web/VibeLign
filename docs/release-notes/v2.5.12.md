## VibeLign v2.5.12 - Windows 정적 HTML 미리보기 빈 화면 수정

### 수정

- Windows에서 `실행해보기`가 `index.html` 정적 앱을 열 때 미리보기 창이 하얀 화면으로 멈추던 문제를 보강했다.
- 정적 HTML 미리보기는 더 이상 임시 `127.0.0.1` HTTP 주소를 WebView2에 직접 로드하지 않고, 실행 대상 HTML 파일의 `file://` 주소를 앱이 발급해 연다.
- 임의 파일 열기를 막기 위해 현재 실행이 발급한 file URL만 허용하고, 미리보기 창 안의 file navigation은 해당 프로젝트 폴더 내부로 제한했다.
- Vite/Next 같은 dev 서버 미리보기는 기존처럼 로컬 `http://localhost` / `127.0.0.1` 주소를 유지한다.

### 검증

- `cargo test --manifest-path vibelign-gui/src-tauri/Cargo.toml commands::run_preview`
- `cargo check --manifest-path vibelign-gui/src-tauri/Cargo.toml`
- `npm --prefix vibelign-gui run build`

### 업그레이드 메모

- v2.5.11에서도 Windows `실행해보기` 팝업이 빈 화면으로 멈췄다면 v2.5.12 설치 후 같은 `index.html` 프로젝트에서 다시 실행하면 된다.
