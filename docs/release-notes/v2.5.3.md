# VibeLign v2.5.3

Released: 2026-06-21

## 한국어

VibeLign v2.5.3은 **보고서 카드뉴스 출력**을 중심으로 다듬은 릴리스입니다. 보고서 내용을 누구나 빠르게 이해할 수 있는 3-6장 요약 카드로 나누고, 사용자가 선택한 CLI 모델별로 카드 문구와 그림 프롬프트, 실제 카드 이미지가 달라지도록 연결했습니다.

- **카드뉴스 초안 모델 선택**: 카드뉴스 companion에서 Local, Claude Code, Codex, Antigravity, OpenCode/DeepSeek 계열을 선택해 초안을 만들 수 있습니다.
- **내용 기반 이미지 생성**: 각 카드의 `visual_prompt`를 바탕으로 카드 본문 칸에 들어갈 SVG asset을 만들고 `.vibelign/reports/card-news/assets/`에 저장합니다.
- **초안과 최종 HTML 표현 일치**: 초안 화면에서도 최종 HTML 카드뉴스처럼 생성 이미지가 카드 안에 표시됩니다.
- **모델별 결과 차이 보존**: 모델이 다시 쓴 문구와 `visual_prompt`가 asset 생성까지 이어져, 모델별 이미지가 같은 placeholder로 보이는 문제를 줄였습니다.
- **LLM용 prompt pack 제공**: 카드뉴스 확정 시 각 LLM에서 바로 재생성·비교할 수 있는 prompt 파일을 함께 저장합니다.
- **Windows 경로 보강**: Windows 절대경로(`C:\Users\...\ .vibelign\reports\card-news\assets\...`)로 저장된 asset도 앱에서 이미지로 렌더링됩니다.
- **로컬 asset 렌더링 안정화**: Tauri `assetProtocol`과 `convertFileSrc()`를 사용해 `unsupported URL` 및 깨진 이미지 아이콘 문제를 방지했습니다.
- **오래된 CLI 안내 개선**: 설치된 `vib`가 `report-card-news` 명령을 모를 때 빈 JSON parse 오류 대신 업데이트 안내를 보여줍니다.

### 확인한 항목

- Python 카드뉴스 asset/Windows 경로 테스트
- GUI 카드뉴스 미리보기 테스트
- `npm run build`
- Tauri debug build

네이티브 Windows/macOS 패키지는 v2.5.3 태그 푸시 후 GitHub Actions runner에서 빌드됩니다.

## English

VibeLign v2.5.3 focuses on **report card-news output**. Reports can now become easy-to-scan 3-6 card summaries, and the selected CLI model can influence the card copy, visual prompt, and generated card image asset.

- **Model selection for card-news drafts**: Local, Claude Code, Codex, Antigravity, and OpenCode/DeepSeek-style providers are available from the card-news companion.
- **Content-aware card image assets**: each card's `visual_prompt` is materialized into an SVG asset under `.vibelign/reports/card-news/assets/`.
- **Draft preview matches final HTML**: generated images appear inside the draft card body slot, not only in the exported HTML.
- **Provider-specific visual variation**: model-authored card copy and `visual_prompt` now flow into asset generation, reducing same-looking placeholder images.
- **Prompt packs for LLM reuse**: finalized card-news exports include prompt files that can be run directly in other LLMs for comparison or regeneration.
- **Windows path handling**: generated assets saved with Windows absolute paths render correctly inside the app.
- **Stable local asset rendering**: Tauri `assetProtocol` and `convertFileSrc()` prevent unsupported local URL and broken image cases.
- **Clear stale-CLI guidance**: outdated installed `vib` binaries now produce an update message instead of an empty JSON parse error.
