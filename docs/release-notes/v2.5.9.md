## VibeLign v2.5.9 — 실행해보기 진입점 생성/복구 보강

기획방에서 AI가 코드를 완성한 뒤 `실행해보기` 버튼이 비활성화되고 “실행 방법을 못 찾았어요”로 막히던 흐름을 고쳤다. 앞으로 작업방 구현 지시에는 바이브라인이 직접 감지할 수 있는 루트 `index.html` 또는 `package.json`의 `dev`/`start` 스크립트를 반드시 만들도록 요구한다.

### Fixed

- **실행 가능한 진입점 누락** — AI 작업 지시문에 실행해보기 감지 조건을 추가했다. 정적 HTML 앱은 프로젝트 루트의 `index.html` 하나만으로 열리게 만들고, Vite/React 같은 앱은 `package.json`의 `dev` 또는 `start` 스크립트를 제공해야 한다.
- **감지 실패 후 막힌 UX** — 실행 화면에서 진입점을 찾지 못했을 때 기획방으로 되돌아가라고만 안내하지 않고, `작업방에서 실행 형태 만들기` 버튼으로 바로 복구 지시를 넘길 수 있게 했다.
- **복구 지시 품질** — 실행 실패 핸드오프 지시문도 루트 `index.html` 또는 `package.json` 실행 스크립트 생성 기준을 포함하도록 보강했다.

### Upgrade Notes

- 기존에 실행 버튼이 비활성화되던 프로젝트는 v2.5.9에서 `실행해보기` 화면의 `작업방에서 실행 형태 만들기`를 눌러 루트 `index.html` 또는 실행 스크립트를 추가하게 하면 된다.
- 새로 기획방에서 만든 작업은 완료 전 실행해보기 감지 대상이 생기도록 지시되므로, Windows에서도 같은 조건으로 직접 미리보기를 열 수 있다.

### Verified

- `npm --prefix vibelign-gui run build`로 GUI production build를 확인했다.
- `git diff --check`로 릴리스 변경의 whitespace를 확인했다.
