## VibeLign v2.5.15 - Windows 미리보기 데드락 근본 수정

### 핵심 수정

- Windows `실행해보기` 미리보기 창이 빈 화면으로 멈추고 앱 전체가 얼어붙던(클릭은 받지만 동작 안 함, 닫기도 불가) 문제를 **근본 원인**에서 수정했다.
- 원인은 미리보기 URL 방식이 아니었다. v2.5.10~v2.5.14는 로딩 주소(localhost → 127.0.0.1 → `file://` → 커스텀 프로토콜 → localhost)만 바꿨지만, 진짜 트리거는 `open_preview` 가 **동기 `#[tauri::command]`** 라는 점이었다. Tauri 공식 문서가 명시하듯 `WebviewWindowBuilder::build()` 를 동기 커맨드에서 호출하면 Windows(WebView2)에서 메인 스레드 재진입 데드락이 난다(tauri#13963: 빈 화면 + 닫기 버튼 무응답). macOS(WKWebView)는 이 재진입이 없어 정상 동작했다.
- `open_preview` 를 `async` 커맨드로 전환해 tokio 풀에서 실행 — 메인 스레드가 WebView2 메시지 펌프를 계속 돌릴 수 있어 데드락이 사라진다.
- 반복 열기 보강: 미리보기 창이 이미 있으면 `destroy()` + `build()` 로 같은 label 을 재생성하지 않고(같은-label 경합, tauri#13969·#9353), 기존 창을 그 URL 로 `navigate` + 포커스해 재사용한다. 두 번째 `미리보기 열기` = 기존 창을 앞으로.

### 검증

- `cargo check --manifest-path vibelign-gui/src-tauri/Cargo.toml`
- `cargo test --manifest-path vibelign-gui/src-tauri/Cargo.toml commands::run_preview`

### 테스트 안내 (Windows 실기기 필수)

- 이 수정은 macOS에서 컴파일·테스트만 확인했다. WebView2 데드락은 Windows에서만 재현되므로, 출시 전 Windows에서 아래를 반드시 확인한다:
  1. `index.html` 만 있는 프로젝트에서 `실행해보기` → `미리보기 열기` → 창에 실제 UI가 뜨고 **앱이 멈추지 않는지**.
  2. 미리보기 창을 **두 번 이상** 열어 같은-label 재생성 경합 없이 기존 창이 앞으로 오는지.
  3. `중지` 버튼이 즉시 실행 상태를 해제하는지.
