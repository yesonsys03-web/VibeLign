## VibeLign v2.5.6 — Intel(x86_64) macOS 빌드 + 자동 업데이트 지원

지금까지 macOS 릴리즈는 Apple Silicon(arm64) 산출물만 제공했다. v2.5.6부터 GUI CI가 Intel(x86_64) macOS 빌드도 함께 만들어 서명·업로드하고, 업데이터 매니페스트(`latest.json`)에 `darwin-x86_64` 항목을 추가한다. Intel Mac 사용자도 이제 첫 설치 이후 인앱 자동 업데이트를 받는다.

### Added

- **Intel macOS 네이티브 빌드** — GUI CI 빌드 매트릭스에 Intel 네이티브 러너(`macos-15-intel`)를 추가했다. PyInstaller로 묶는 `vib-runtime`은 호스트 아키텍처로만 빌드되어 크로스컴파일이 불가능하므로, x86_64 산출물은 Intel 네이티브 러너에서 빌드한다. (이전 Intel 러너 `macos-13`은 2025-12 은퇴.) 릴리즈 에셋으로 Intel `.dmg`와 업데이터 아카이브(`vibelign-gui_x64.app.tar.gz`)가 추가된다.
- **Intel 자동 업데이트** — 릴리즈 워크플로가 러너 아키텍처에 맞는 업데이터 platform 키(`darwin-aarch64` / `darwin-x86_64`)를 생성하고, 두 macOS 매니페스트 조각을 병합해 `latest.json`에 두 아키텍처를 모두 싣는다. Intel 앱이 자기 아키텍처용 서명 아카이브를 받아 검증한다.

### Changed

- **macOS 업데이터 아카이브 유일화** — arm64/Intel 두 빌드의 `.app.tar.gz`가 동일한 이름이라 릴리즈 에셋이 충돌하던 문제를 막기 위해 아키텍처 라벨을 붙여 분리했다(`vibelign-gui_aarch64.app.tar.gz` / `vibelign-gui_x64.app.tar.gz`). 업데이터 서명은 파일 내용 기준이라 이름 변경은 검증에 영향이 없다.
- **Intel 빌드 실패 격리** — Intel(`macos-15-intel`) 빌드가 실패해도 arm64/Windows 릴리즈 발행을 막지 않도록 릴리즈 잡이 Intel 산출물 누락을 건너뛴다.

### Verified

- 워크플로 YAML 파싱과 빌드 매트릭스 표현식, 매니페스트 병합/publish 글롭 구조를 로컬에서 확인했다. 실제 Intel `.dmg` 생성과 인앱 자동 업데이트 적용은 태그 빌드 산출물에서 확인 대상이다.
