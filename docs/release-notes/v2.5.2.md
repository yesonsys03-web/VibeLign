# VibeLign v2.5.2

Released: 2026-06-19

## English

VibeLign v2.5.2 unifies the report-export option panel under the code-explorer's side-button design language.

- Replaced the report type (Work/Proposal/Result/Doc) and format (HTML/PDF/Word/PPT) radios with full-width side buttons that match the code-explorer sidebar (dark fill + left accent bar when selected).
- Turned the "page numbers" and "AI tone polish" checkboxes into the same side-button toggles (with a ✓ indicator).
- Aligned the design-theme, author, and font selectors to the brutalist `input-field` style (black border + shadow + full width) and added a label header to each group.
- Fixed the "PPT file" button being clipped — labels weren't picking up `.btn`'s display, so they flowed horizontally; an explicit `display:flex` restores full-width vertical stacking.
- Radios and checkboxes keep a visually-hidden native `input`, preserving native toggle, keyboard, accessibility, and tests. Verified with TypeScript, ESLint, the Vite build, and the GUI test suite.

## 한국어

VibeLign v2.5.2는 보고서 내보내기 옵션 패널을 코드탐색 화면의 사이드 버튼 디자인 언어로 통일한 릴리스입니다.

- 보고서 종류(업무/제안/결과/문서)와 형식(HTML/PDF/Word/PPT) 라디오를 코드탐색 사이드바와 같은 풀폭 세로 버튼(선택 시 다크 배경 + 좌측 액센트 바)으로 교체했습니다.
- 페이지 번호·AI 어조 다듬기 체크박스를 동일한 사이드 버튼식 토글(✓ 표시)로 통일했습니다.
- 디자인 테마·작성자·폰트 종류 컨트롤을 브루탈리즘 `input-field` 스타일(검정 테두리 + 그림자 + 풀폭)로 맞추고 그룹마다 라벨 헤더를 추가했습니다.
- `<label>`이 `.btn`의 display 를 못 받아 가로로 흐르며 잘리던 "PPT 파일" 버튼을 `display:flex` 명시로 풀폭 세로 적층되도록 수정했습니다.
- 라디오·체크박스는 네이티브 `input`을 시각적으로만 숨겨 네이티브 토글·키보드·접근성·테스트를 모두 보존했습니다. TypeScript, ESLint, Vite build, GUI 테스트로 확인했습니다.
