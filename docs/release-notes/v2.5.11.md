## VibeLign v2.5.11 - Windows 실행해보기 WebView2 빈 화면 보강

Windows에서 `실행해보기` 팝업은 뜨지만 `index.html`이 빈 화면으로 멈추던 WebView2 경로를 보강했다. 브라우저에서 파일이 정상 동작하는 경우에도, 앱 내부 미리보기 창의 네비게이션 필터가 WebView2 내부 부팅 URL을 막으면 렌더러가 빈 화면에 머물 수 있었다.

### Fixed

- **WebView2 내부 네비게이션 차단** - 미리보기 보안 필터가 로컬 `http(s)`만 허용하던 상태에서 `about:blank`, `data:`, `blob:` 같은 WebView2 내부/프레임 네비게이션을 취소할 수 있던 문제를 수정했다.
- **빈 화면 렌더러 재사용** - 기존 미리보기 창이 있으면 `navigate()`로 재사용하지 않고 창을 새로 만들어, 한 번 멈춘 WebView2 렌더러 상태를 다음 실행에 재사용하지 않게 했다.
- **외부 이동 차단 유지** - `https://evil.example.com`, `file://` 같은 외부/로컬 파일 직접 접근은 계속 차단한다.

### Upgrade Notes

- v2.5.10에서도 Windows 미리보기 팝업이 빈 화면으로 멈췄다면 v2.5.11 설치 후 같은 프로젝트에서 `실행해보기`를 다시 눌러 확인하면 된다.
- 웹브라우저에서 열리는 `index.html`은 그대로 두고, VibeLign 앱 내부 팝업 로딩 경로만 보강했다.
