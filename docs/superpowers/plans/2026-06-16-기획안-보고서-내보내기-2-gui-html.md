> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 기획안 탭(`PlanDocView`)에서 저장된 기획안을 골라 **종류(업무/제안/결과)** 를 선택하고 `vib report` CLI 를 호출해 HTML 보고서를 생성한 뒤, **앱 내 미리보기**(iframe)와 **파일 열기**를 제공한다.

**Architecture:** 증분 1 의 `vib report` CLI 를 그대로 호출한다. 새 Rust 커맨드는 필요 없다 — 기존 `run_vib(args, cwd, env)` 브리지(`vib_bridge.rs`)가 임의의 vib 인자를 실행하고 `{ok,stdout,stderr,exit_code}` 를 돌려준다. 따라서 이 증분은 **프론트엔드 전용**이다: ① TS 래퍼(`report.ts`)가 `run_vib` 로 CLI 를 호출하고 `--json` stdout 을 파싱한 뒤 생성된 HTML 파일을 기존 `read_file`(loadDoc)로 다시 읽어 미리보기 내용을 함께 반환, ② `ExportReportModal` 이 종류 선택·생성·미리보기·파일 열기를 담당(`GuardResultModal` 패턴), ③ `PlanDocView` 가 기획안별 버튼으로 모달을 연다.

**Tech Stack:** TypeScript, React, Vitest + @testing-library/react, `@tauri-apps/plugin-opener`. 신규 외부 의존성 없음. Rust 변경 없음.

**상위 설계 문서:** `docs/superpowers/specs/2026-06-15-기획안-보고서-내보내기-design.md` (§7 HTML 단일 소스, §8 UI/UX). PDF 무인 생성(§13.1 a/b/c)은 **이 증분 범위 밖** — 계획 2b 에서 처리. 이 증분의 PDF 는 "미리보기/파일 열기 → 브라우저 Cmd+P" 로 사용자가 직접.

**실행 브랜치:** `feat/report-export-gui` (base = `feat/report-export` ea41e77). 증분 1 의 `vib report` CLI 가 이미 이 브랜치에 포함돼 있다.

```ts
import { runVib } from "./core";
import { loadDoc } from "../docs";

export type ReportType = "work" | "proposal" | "result";

export type ReportResult =
  | { ok: true; path: string; reportType: string; html: string }
  | { ok: false; error: string };

/** 절대 보고서 경로를 프로젝트 루트 기준 상대경로로 변환한다.
 *  read_file(loadDoc) 가 root-상대 경로를 기대하기 때문. 루트 밖이면 원본 유지. */
export function toProjectRelative(cwd: string, absPath: string): string {
  const norm = (p: string) => p.replace(/\\/g, "/").replace(/\/+$/, "");
  const root = norm(cwd);
  const p = norm(absPath);
  if (p === root) return "";
  if (p.startsWith(root + "/")) return p.slice(root.length + 1);
  return absPath;
}

export async function generatePlanningReport(
  cwd: string,
  planPath: string,
  reportType: ReportType,
): Promise<ReportResult> {
  const res = await runVib(
    ["report", planPath, "--type", reportType, "--format", "html", "--json"],
    cwd,
  );

  let parsed: { ok?: boolean; path?: string; report_type?: string; error?: string };
  try {
    parsed = JSON.parse(res.stdout.trim());
  } catch {
    return {
      ok: false,
      error: res.stderr.trim() || res.stdout.trim() || "보고서 생성에 실패했어요.",
    };
  }

  if (!parsed.ok || !parsed.path) {
    return { ok: false, error: String(parsed.error ?? "보고서 생성에 실패했어요.") };
  }

  const rel = toProjectRelative(cwd, parsed.path);
  const doc = await loadDoc(cwd, rel);
  return {
    ok: true,
    path: parsed.path,
    reportType: String(parsed.report_type ?? reportType),
    html: doc.content,
  };
}
```
