# VibeLign — AI 코딩의 안전벨트

> *"AI 코딩은 빠르다. 하지만 안전장치가 없으면, 빠른 만큼 빠르게 무너진다."*

바이브 코딩을 한 번이라도 해본 사람이라면 이 장면이 낯설지 않을 것이다.

작은 기능 하나만 추가해 달라고 했더니 AI가 **파일 전체를 새로 써버린다.** 정신을 차려보니 모든 코드가 `main.py` 한 파일에 1,000줄 넘게 뭉쳐 있다. 분명 잘 돌아가던 화면이, AI가 엉뚱한 파일을 건드린 뒤로 먹통이 됐다. 되돌리고 싶은데 Git을 모르니 어디서부터 손대야 할지 막막하다.

**VibeLign은 바로 이 순간들을 위해 만들어졌다.** AI 코딩 도구(Claude Code, Cursor, Codex, OpenCode 등)가 일으키는 사고로부터 내 프로젝트를 지켜주는 **안전망(safety net)**이다. 터미널에서 쓰는 CLI와 클릭만으로 쓰는 데스크톱 앱(macOS·Windows), 두 가지 형태로 제공된다.

VibeLign의 화면은 바이브 코딩의 흐름을 그대로 본떠 **세 단계**로 짜여 있다. **📋 기획 → ⚙️ 개발 → 🛡️ 유지보수.** 아이디어를 떠올린 순간부터, AI에게 일을 시키고, 망가진 걸 되돌리는 순간까지 — 한 도구가 처음부터 끝까지 함께 간다.

## 📋 기획 — 코딩하기 전에, 먼저 생각을 정리한다

대부분의 바이브 코딩 사고는 코드가 아니라 **막연한 한 줄의 지시**에서 시작된다. "카페 예약 앱 만들어줘"라고만 던지면 AI는 빈칸을 제멋대로 상상해 채운다. 좋은 결과는 좋은 기획에서 나오지만, 코딩을 막 시작한 사람("코알못")에게 기획서를 직접 쓰라는 건 또 다른 벽이다.

VibeLign의 **기획방**은 이 벽을 없앤다. 채팅창에 평소 말로 아이디어를 던지면 쉬운 질문 서너 개가 돌아온다. "누가 쓰나요?" "가장 먼저 하고 싶은 일은?" "이번엔 빼도 되는 건?" 답만 하면, 세 명의 AI 페르소나가 등장해 서로 다른 관점으로 기획을 다듬는다.

- **설계자 클로이**가 흩어진 생각을 기획안 초안으로 구조화하고,
- **검토자 지오**가 빠진 요구사항과 모순을 짚어내고,
- **탐색자 미나**가 "처음엔 이만큼만 만들자"며 현실적인 MVP로 범위를 좁힌다.

각 페르소나는 사용자가 이미 터미널에 로그인해 둔 공식 AI 도구(Claude·Codex·Gemini 계열)와 연결돼 있어, **여러 AI를 한 팀처럼** 부릴 수 있다. 마크다운 문법을 한 글자도 몰라도, 대화가 끝나면 **AI가 곧바로 이해하는 기획안**이 한 편의 문서로 저장된다. 이 파일을 그대로 코딩 AI에게 건네면 작업의 첫 단추부터 정확하게 끼워진다.

## ⚙️ 개발 — AI에게 안전하게 일을 맡긴다

기획안이 나오면 본격적인 코딩이 시작된다. 이 단계에서 VibeLign은 AI가 사고를 *치기 전에* 선을 그어 준다.

- **앵커(Anchor)** — "여기, 이 구역만 고쳐"라고 AI에게 안전 구역을 표시해 엉뚱한 곳을 건드리지 못하게 한다.
- **코드 탐색·문서 보기** — 내 프로젝트가 어떻게 생겼는지, AI가 무엇을 어디에 썼는지 한눈에 훑어볼 수 있다. 코드를 몰라도 구조가 보인다.

핵심은 AI의 속도를 늦추지 않으면서, AI가 손댈 수 있는 범위를 사용자가 쥐고 있게 하는 것이다.

## 🛡️ 유지보수 — 망가져도, 언제든 되돌린다

VibeLign의 진짜 힘은 사고가 났을 때 드러난다. 이 단계의 핵심은 게임의 '세이브 / 불러오기'만큼 단순하다. Git을 전혀 몰라도 된다.

```
AI에게 시키기 전   →  vib checkpoint "작업 전"   # 저장
AI가 망가뜨렸다면   →  vib undo                  # 되돌리기
잘 됐으면          →  vib checkpoint "완료"      # 다시 저장
```

위험한 작업 전에 1초 만에 저장하고, 일이 틀어지면 1초 만에 직전 상태로 돌아간다. 보스전 직전에 세이브하는 것과 똑같은 감각이다. 여기에 더해,

- **진단** — `vib doctor`·`vib guard`가 프로젝트가 망가지지 않았는지 자동으로 점검한다.
- **시크릿 차단** — API 키, 토큰, `.env` 파일이 실수로 커밋되는 것을 막는다. (한번 새어 나간 비밀은 되돌릴 수 없으니까.)
- **에러로그** — 무엇이 어디서 잘못됐는지 한 화면에 모아 보여주고, 필요하면 버튼 한 번으로 개발자에게 신고까지 보낸다.
- **세션 인계** — 토큰 한도에 부딪히거나 AI 도구를 바꿀 때, `vib transfer`가 프로젝트의 맥락을 파일 하나로 정리한다. 새 AI에게 "이 파일 먼저 읽어"라고만 하면 작업이 그대로 이어진다.

## 누구를 위한 도구인가

VibeLign은 코딩을 막 시작한 사람부터 AI에게 작업을 위임하는 숙련 개발자까지 모두를 위한 것이다. 설치는 한 줄(`pip install vibelign`), 시작도 한 줄(`vib start`)이면 끝난다. 특정 AI에 묶이지 않고 어떤 AI 코딩 도구와도 함께 쓸 수 있다.

바이브 코딩의 매력은 속도다. VibeLign은 그 속도를 늦추지 않으면서, 빠르게 달리다 넘어졌을 때 다시 일어설 수 있는 **안전벨트**를 채워준다. 기획부터 개발, 유지보수까지 — AI가 만든 것을 AI가 부수더라도, 당신은 언제든 직전으로 돌아갈 수 있다.
