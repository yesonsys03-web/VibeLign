# 마스코트 가이드 레이어 — 네이티브 포팅 계획

> 작성일: 2026-06-13 · 근거 목업: `plans/2026-06-10-강PD-가이드레이어-목업.html` (v6)
> 갭 분석 결론: 기능·데이터는 실제 앱이 이미 완성. 빠진 것은 **마스코트(🧭) 말풍선 캐릭터 레이어**뿐.

## 1. 목표와 범위

목업의 "게임처럼 즐기는" 차별점 = **마스코트 말풍선 + 코치마크**를 실제 React 컴포넌트에 입힌다.
정적 HTML 목업을 앱에 넣지 않는다(시뮬레이션이라 진짜 동작과 단절). 목업은 **디자인 스펙·QA 기준**으로만 보관.

### 포함 (A 구간 — 실제 손댈 것)
1. 가이드 스트립을 마스코트(🧭)+말풍선 톤으로 (단계 전환 "뽁" pop 포함)
2. 온보딩 환영 인사 말풍선
3. 온보딩 `+` 버튼 코치마크 (단계 추적: 폴더→입력, 호버 팁, 펄스링)
4. MCP "전체 선택" 설명 말풍선
5. 첫 사이클 완주 축하에 말풍선 톤 적용 (로직은 이미 있음 → 스킨만)

### 제외 (건드리지 않음)
- **C 구간 (LLM/실데이터)**: 기획방 페르소나 대화, 기획안, 작업 지시문, 문서·진단·백업·에러로그 콘텐츠 —
  이미 LLM(opus)·실백엔드로 "주제별 구체화"가 진짜로 일어남. 목업의 키워드 하드코딩(`topicProfile`)을
  옮기면 **퇴보**. 절대 포팅 금지.
- 실행 미리보기 주제별 accent 색(`topicAccent` 10종): 순수 장식 → 이번 범위 밖(향후 선택).
  (정 필요하면 기획안 계약에서 도메인 1개를 뽑아 테마색을 정하는 가벼운 방식으로, 별도 트랙.)

### 비목표
- 새 페이지·새 탭 추가 없음. 기존 컴포넌트 내부 시각 톤만 변경.
- 가이드 단계 로직(`guide.ts inferStep`/`useGuide.ts`) 변경 없음 — 신호·단계 판정은 그대로 둔다.

## 3. 공통 디자인 자산 (먼저 1회 추가)

전역 CSS(예: `src/index.css` 또는 앱 전역 스타일)에 keyframes·말풍선 클래스 추가.
목업의 값을 그대로 차용 — 앱 전체 크림 네오브루탈 테마(#FEFBF0, 2px 검정 테두리, box-shadow)와 일치.

- `@keyframes popbubble`: `scale(0) rotate(-7deg)` → `scale(1.14)` → `scale(.95)` → `scale(1)` (오버슈트 바운스, `cubic-bezier(.18,1.5,.4,1)`, ~.42s)
- `@keyframes pulsering` / `@keyframes nudge`: `+` 끌어당김
- 색: 말풍선 배경 `#FFF6DC`, 테두리 `#1A1A1A`, 강조 `#7C2D12`, 마스코트 원형 `#FBBF24`, 축하 `#E8F6EC`/`#166534`
- 마스코트 아바타: 34px(스트립)·46px(온보딩 환영) 원형, 2.5px 테두리, `2px 2px 0` 그림자

> 구현 메모: React에서는 keyframes를 전역 CSS에, 정적 스타일은 클래스(`.mascot`/`.gbubble`/`.bubble`/`.hovertip`/`.tip-bubble`)로,
> 동적 값(현재 단계 등)은 인라인 style로. 목업의 인라인 스타일 덩어리를 클래스로 정리하며 옮긴다.
