# 모션 차원 (Motion Dimension) 구현 계획 (첫 슬라이스)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 각 디자인 스타일에 고유한 모션 성격을 베이크 — 목업 프롬프트엔 CSS-only 모션으로(프리뷰에서 실제 동작), 코딩 핸드오프엔 풀 모션 어휘로 전달한다. 첫 슬라이스는 네오브루탈리즘 1종.

**Architecture:** `StyleSpec`에 optional `motion: { tokens:{duration,easing}, recipe }` 추가(TS+Rust, 하위호환). 모션 토큰은 색 토큰처럼 `--dur`·`--ease` CSS 변수로 결정적 주입(`style_to_css_vars`). `build_mockup_prompt`는 모션 섹션에 "CSS 전환·@keyframes·:hover만, JS 금지, reduced-motion 존중" 제약을 붙이고, `buildPlanningWorkInstruction`은 풀 어휘 "[모션 가이드]"를 핸드오프에 싣는다. validator는 CSS 모션을 막지 않으므로 안전.

**Tech Stack:** Rust(serde), React/TypeScript(Vitest). 기반: 디자인 프리뷰 PR A(이미 머지된 `design_preview.rs`·`styles.ts`·`planningInstruction.ts`·`DesignPreview.tsx`).

**선행 의존:** PR A 완료(StyleSpec, tokens_to_css_vars, build_mockup_prompt, generate_design_mockup, buildPlanningWorkInstruction design 섹션, DesignBinding). 본 계획은 그 위에 motion을 얹는다.

**범위 밖(후속, design §7):** 나머지 4스타일 motion · 모션 강도 축 · 갤러리 모션 뱃지.

```rust
/// 색 토큰 :root 블록에 motion 토큰(--dur/--ease)을 합류. motion 없으면 색 토큰만.
pub(crate) fn style_to_css_vars(style: &StyleSpec) -> String {
    let mut s = tokens_to_css_vars(&style.tokens);
    if let Some(m) = &style.motion {
        if let Some(pos) = s.rfind('}') {
            s.insert_str(pos, &format!("--dur:{};--ease:{};", m.tokens.duration, m.tokens.easing));
        }
    }
    s
}
```
