## 배경 (강 PD 지적 5종)

초보자가 "어디서 뭘 할지" 모름. 상단 탭이 평평하고(`App.tsx:291~333`) 한자어/영어 혼재.
A안은 **탭 구조를 바꾸지 않고** 인지 부담만 낮춘다.

현재 메인 탭:
```
홈 │ Doctor │ 메뉴얼 │ 폴더열기 │ DOCS VIEWER │ CODE EXPLORER │ BACKUPS │ 에러로그   (…우측: 프로젝트명↩)
```

목표 메인 탭(네비게이션만 남김) — **결정 확정 2026-06-10**:
```
홈 │ 진단 │ 문서 │ 코드탐색 │ 백업 │ 에러로그        (…우측 유틸 클러스터: 사용법 · 폴더열기 · 설정 · 프로젝트명↩)
```

## Part 2 — 유틸을 메인 탭에서 분리 (저위험 재배치)

강 PD: "폴더열기 같은 유틸을 메인 탭에서 분리." **드롭다운 신규 제작은 금지** — 코드에 오버플로/팝오버 컴포넌트가 0건이라 새로 만들면 A안 유일의 net-new 위험 요소가 된다.

**채택: 우측 유틸 클러스터로 재배치** (`App.tsx:316` 의 `<div style={{flex:1}}/>` 스페이서 오른쪽 = 이미 프로젝트명 버튼이 사는 영역). 좌측=네비게이션 탭, 우측=유틸로 시각 분리. 신규 컴포넌트 없음.

- `폴더열기` 버튼(`App.tsx:301~303`) → 우측 클러스터로 이동 (페이지 전환 아니라 `openFolder` 액션 그대로).
- `메뉴얼`(`App.tsx:298~300`) → **✅ 우측 "사용법" 버튼으로 확정.** `page==="manual"` 은 `<Home initialView="manual_list">` 렌더(`App.tsx:339`) — 라우팅/렌더 로직 그대로, 버튼 위치+라벨("사용법")만 이동.
- (선택) 우측 클러스터에 옅은 구분선/라벨로 "도구" 묶음 시각화.

## Part 3 — 상태 가시화 (신규 데이터 아님, 가시성만)

⚠️ **메모리 정정:** "기존 진행률 대시보드 재활용"은 부정확. 코드에 대시보드 컴포넌트는 **없다**(grep 0건) — `plans/VibeLign-기획방개선-기획안.md` 의 *계획*이었을 뿐.

단, 신호 자체는 **이미 존재하고 이미 Home 에 렌더된다**: `watchOn`/`hasCheckpoint`(`Home.tsx:152,154,179`), guard 상태, `planningPending`. 따라서 빈틈은 **데이터가 아니라 "앱 어디서나 보이는 전역 가시성"** 이다.

**채택(저위험) — ✅ 탭바 *아래* 얇은 가로 줄로 확정:** **컴팩트 상태 스트립** — 기존 신호 재사용.
- 감시: 켜짐/꺼짐 (watchOn) + 꺼짐이면 클릭으로 켜기
- 마지막 체크포인트 유무 (hasCheckpoint)
- guard: 정상/이슈 (이슈면 Doctor=점검 으로 점프, 기존 `openDoctorFromGuardIssue` 재사용)
- 기획 진행 중이면 표시 (planningPending)

신규 백엔드/명령 없음. 신규 컴포넌트는 작은 표시줄 1개만(데이터는 전부 기배선).

**스코프 가드:** 새 대시보드 페이지를 만들지 말 것 — 그러면 "며칠" 예산이 깨진다. 한 줄짜리 상태 표시까지만.
