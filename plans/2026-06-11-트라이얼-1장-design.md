> 작성 2026-06-11 · 브레인스토밍 세션에서 사용자와 단계별 승인 완료

## 1. 확정 결정 (브레인스토밍 승인 이력)

| 결정 | 선택 | 기각 |
|---|---|---|
| 1차 독자 | **C: 둘 다** — 설치한 초보자(연습장) + 설치 전 잠재 사용자(데모) | — |
| 충돌 시 우선 | **A: 정확성 우선** — 카피는 실제 UI와 한 글자까지 동일(`guide.ts` JOURNEY_STEPS 단일 소스에서 복사), B(전달력)는 "정확한 것을 보기 좋게"로 달성 | B 우선(부정확 데모는 설치 후 신뢰 훼손 — 비용이 마케팅 효과보다 큼) |
| 시나리오 범위 | **C: 1장(가이드 루프+지문) + 2장 예고편** — 말풍선 ~11개 한 호흡, 챕터 구조로 확장 여지 | A(예고 없음) / B(계약 동시 — 분량 2배 + 육안 미검증 화면 박제 리스크) |
| 구현 방식 | **접근 1: 신규 작성** — 기존 목업은 레이아웃 참고만 | 접근 2(목업 개조 — 시연 구조라 게이팅 개조 비용 ≥ 신규, 낡은 카피 누락 위험) / 접근 3(인앱 코치마크 — 북극성이나 standalone 요구 불충족, 후속) |

포지셔닝: **"앱 복제"가 아니라 "체험 시뮬레이션"** — 핵심 동선만 본뜬 단순화된 무대에서 가짜 상태로 여정을 연출. 드리프트 리스크(복제 GUI는 실제 앱 변경마다 낡음)를 단순화로 흡수.

## 2. 파일·구조·충실도 규칙

```
docs/tutorial/vibelign-trial-ch1.html   ← 신규 단일 파일 (React 18 CDN + Babel standalone)
```

- 파일명 영문(B 용도 — URL 공유·랜딩 임베드), 위치는 배포 자산용 `docs/tutorial/`(신규 디렉터리).
- 구조 분리: `CHAPTERS`/`TRIAL_STEPS`(말풍선·대상·완료 조건) + `simState`(가짜 상태) + 화면 컴포넌트. 2장 추가 = 데이터 추가.
- **충실도 규칙(A 우선의 구체화)**:
  - 본뜨는 것: 두 탭바, GuideStrip(13px·#FBBF24, affordance·비상구 문구 그대로), 홈 SimpleHome 3블록("상태 확인하기" 라벨 동일), 허브 카드 3장 배지, 백업 체크포인트 버튼, 코드탐색 지시문 복사. 카피 전부 `guide.ts` JOURNEY_STEPS + 실제 컴포넌트 라벨에서 복사, 파일 상단 주석에 출처·동기화 의무.
  - 생략: 설정·진단·로그·고급 카드·명령어 사전 내용. 첫 말풍선에서 "연습장이라 일부 화면은 간단해요" 고지.
  - **연습 전용 장치의 시각 분리**: 실제 앱에 없는 시뮬레이션 버튼(⏩)은 보라색 + "연습 전용" 라벨 — 초보자가 실제 앱에서 찾는 사고 방지.

## 3. 트라이얼 단계 시퀀스 (말풍선 11개 + 완료)

| # | 말풍선 | 사용자 행동 | 화면 반응 |
|---|---|---|---|
| 0 | 환영 + "진짜 프로젝트는 안 건드려요" + 간소화 고지 | [시작하기] | — |
| 1 | "스트립이 지금 할 일을 알려줘요" (2️⃣) | 스트립 [기획으로 이동 →] | 기획방(간이) |

| 3 | "AI에게 시키기 **전에** 안전 저장" (3️⃣) | 백업 탭 → [체크포인트 저장] | 스트립 4️⃣, 카드 배지 변화 |
| 4 | "지시문을 복사해 AI 도구에" (4️⃣) | 코드탐색 → [지시문 복사] | 복사 토스트 |

| 6 | "탭을 오가면 변경을 알아채요" | 아무 탭 클릭 | 스트립 5️⃣ 자동 전환 |
| 7 | "약속한 범위만 고쳤는지 확인" (5️⃣) | 홈 → [상태 확인하기] | pass → 6️⃣ |
| 8 | **지문 미션** "이미 고친 파일을 또 고치면?" | ⏩ [같은 파일을 다시 수정했어요] + 탭 전환 | **6️⃣→5️⃣ 복귀** — "거짓 '검증 완료'가 없어요" |

## 4. 시뮬레이션 상태 모델

실제 `inferStep`의 축소 미러 — 신호 이름을 실제와 같게(코드가 곧 교보재):

```js
simState = {
  hasPlanDoc: false,      // 말풍선 2에서 true
  hasCheckpoint: false,   // 말풍선 3에서 true
  diskFiles: [],          // 실제 디스크(AI가 고친 것) — [{ path, rev }], rev = 지문의 교육용 단순화
  changedFiles: [],       // 앱이 마지막 조회로 아는 것 — 탭 전환에서 diskFiles와 동기화
  baseline: [],           // 체크포인트 시점 스냅샷(path+rev)
  guardOk: false,
  page: "home",
}
```

- 단계 추론 = 실제 규칙: 기획안 없음→2, 체크포인트 없음→3, baseline 대비 변경 0→4, 변경 있음→5, 변경+guardOk→6.
- **관찰 동기화(재조회 미러)**: sim을 `diskFiles`(실제 디스크)와 `changedFiles`(앱이 아는 것)로 분리 — 실제 앱이 탭 전환 시 changed-set을 재조회하듯(가이드 v6 spec §4-6), `setPage`에서 `changedFiles = diskFiles` 동기화. `inferStep`은 changedFiles만 본다. ⏩ 직후 즉시 알아채게 만들면 "탭을 눌러보세요 — 변경을 알아채요"(말풍선 6)가 거짓이 되기 때문 — 스트립이 탭 전환 순간에 바뀌어야 말풍선 텍스트와 화면이 일치한다.
- **지문 미션 재현**: ⏩ 재수정은 항목 추가가 아니라 **기존 항목 rev +1** — "파일 목록은 그대로인데 알아챈다"가 화면에 그대로. 말풍선 8이 실제와의 차이를 한 줄로 설명("실제 앱은 수정 시각·크기로 알아채요").
- guardOk 리셋: rev 변화 감지 시 false — 지문 리셋 규칙의 미러. 리셋은 **관찰 시점(setPage 동기화)**에 일어난다(검증 시점 지문 vs 현재 지문 비교라는 실제 동작과 동일). 6️⃣→5️⃣ 복귀가 여기서 나옴.

## 5. 말풍선 엔진 (코치마크)

- `TRIAL_STEPS[i] = { target, text, advanceOn: "click:<id>" | "state:<조건>" }` — 전진은 대상 클릭(1~5·7·9·10) 또는 상태 변화 관찰(6·8).
- 하이라이트: 전체 디밍 + 대상 컷아웃(box-shadow 트릭) + 말풍선은 `getBoundingClientRect` 기준 배치(창 크기 무관).
- 클릭 게이트: 디밍이 대상 외 클릭 흡수 — **비상구 버튼만 예외**(보조 말풍선, 진행 유지).
- 진행 UI: "3 / 11" + [건너뛰기](확인 모달 없이 완료 화면으로 — 선형 강제 금지 정신).
- 재시작: [처음부터 다시] = simState 초기화. **localStorage 미사용** — 연습장은 매번 깨끗하게.

## 6. 엣지 · 규칙

1. 데스크톱 기준 — 모바일이면 "PC에서 봐주세요" 한 줄(B 용도로 폰에서 열릴 수 있음).
2. CDN 의존(React 18 + Babel standalone) — 오프라인 미동작을 파일 주석에 명시. 인라인 번들화는 후속.
3. **카피 드리프트 방어**: 상단 주석 "카피 출처: guide.ts JOURNEY_STEPS·SimpleHome·GuideStrip (2026-06-11 기준) — 실제 UI 변경 시 동기화" + 각 카피 출처 표시. 자동 대조 스크립트는 후속이되, 구현 검증에서 grep 1회 대조는 수행(§7).
4. 2장 자리: `CHAPTERS`에 `{ locked: true, teaser: "AI와의 약속 — 작업 계약" }` — 완료 화면 예고편의 데이터 소스.

## 7. 검증 방침

- 단일 HTML — tsc·vitest 없음. **브라우저 육안 체크리스트**: 11단계 완주 / 건너뛰기 / 지문 미션 6️⃣→5️⃣ 재현 / 비상구 예외 클릭 / 재시작 초기화.
- **카피 grep 대조(유일한 자동 게이트)**: `guide.ts`의 shortAction·affordance·비상구 문자열이 HTML에 그대로 존재하는지 셸에서 확인.

## 8. 비목표 (명시)

- 2장(작업 계약) 콘텐츠 — 예고편만, 본편은 계약 UI 육안 검증 후
- 인앱 코치마크(북극성) — 별도 트랙
- 카피 자동 대조 스크립트(CI) / HTML 인라인 번들화(오프라인) — 후속
- 모바일 대응 — 안내 한 줄만

## 9. 구현 순서 제안 (계획서에서 상세화)

1. 뼈대 + simState·추론 미러 + 화면 골격(탭바·스트립·홈·백업·코드탐색 간이) — 카피 복사
2. 말풍선 엔진(데이터·디밍·컷아웃·게이트·진행·건너뛰기)
3. 지문 미션·축하·완료 화면(2장 예고)·비상구 예외
4. 검증(카피 grep + 육안) + push
