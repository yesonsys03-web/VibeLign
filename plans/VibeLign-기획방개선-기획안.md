# VibeLign 기획방 개선 기획안

*작성일: 2026-06-04*
*목적: 기획방(Planning Room)의 활용도를 높이기 위한 3가지 핵심 개선*

---

## 현재 상태 요약

기획방은 사용자의 아이디어를 3명의 AI 페르소나(클로이·지오·미나)와 채팅으로 구체화해서 `plans/*.md` 기획안 문서를 생성하고, Code Explorer의 AI 구현 작업으로 연결하는 기능.

현재 잘 동작하지만, 기획안이 만들어진 후 **"그래서 얼마나 됐어?"를 추적하는 방법이 없고**, 채팅에서 나온 **중요 결정이 휘발**되고, **코알못한테는 페르소나 선택/모드 설정이 복잡**함.

---

## 개선 1: 기획 진행률 대시보드

### 문제
기획안에 "핵심 기능 5개"를 적었는데, 지금 몇 개가 구현됐는지 수동으로 확인해야 함. 이 세션 초반에 "기획안 어디까지 했나 체크해줘"를 수동으로 요청한 것이 바로 이 문제.

### 해결
기획안의 항목들을 코드(앵커, 커밋, 파일)와 자동 대조해서 진행률 표시.

### 구체적 구현

**데이터 흐름:**
```
plans/*.md 파싱
  → 핵심 기능 목록 추출 ("핵심 기능:" 섹션)
  → 각 기능 키워드를 anchor_meta.json의 intent/aliases와 매칭
  → 매칭되면 "구현됨", 안 되면 "미구현"
  → 진행률 = 매칭 수 / 전체 수
```

**GUI 표시:**
```
┌─────────────────────────────────┐
│ 📋 예약 앱 기획안               │
│                                 │
│ ████████░░░░░░░ 60% (3/5)      │
│                                 │
│ ✅ 로그인 기능                  │
│ ✅ 예약 생성                    │
│ ✅ 예약 목록 보기               │
│ ⬜ 알림 발송                    │
│ ⬜ 관리자 대시보드              │
│                                 │
│ [상세 보기]  [기획방으로 돌아가기]│
└─────────────────────────────────┘
```

**위치:** Home 화면에 "기획 진행률" 카드 추가. 또는 기존 기획방 페이지 상단에 진행률 바.

**매칭 로직:**
1. 기획안의 기능 항목 텍스트 추출
2. 텍스트를 토큰화
3. anchor_meta.json의 intent/aliases/description과 토큰 매칭 (patch_suggester의 scoring 재사용 가능)
4. 매칭 점수 임계값 이상이면 "구현됨"
5. 추가: 최근 커밋 메시지에서도 매칭 (커밋에 "로그인 기능 구현" 있으면 매칭)

**갱신 주기:**
- watch가 돌아가는 동안 anchor_meta 갱신될 때마다 자동 재계산
- GUI에서 수동 새로고침도 가능

**데이터 저장:**
```json
// .vibelign/planning/<session_id>/progress.json
{
  "plan_file": "plans/예약-앱-만들고-싶어.md",
  "total_features": 5,
  "completed": 3,
  "features": [
    {"name": "로그인 기능", "status": "done", "matched_anchor": "LOGIN_HANDLER", "matched_commit": "a1b2c3d"},
    {"name": "예약 생성", "status": "done", "matched_anchor": "RESERVATION_CREATE"},
    {"name": "예약 목록 보기", "status": "done", "matched_anchor": "RESERVATION_LIST"},
    {"name": "알림 발송", "status": "pending"},
    {"name": "관리자 대시보드", "status": "pending"}
  ],
  "last_updated": "2026-06-04T10:30:00"
}
```

### 작업 범위

| 항목 | 위치 | 예상 |
|------|------|------|
| 기획안 md 파싱 (기능 목록 추출) | Python `vibelign/core/planning_progress.py` (신규) | 반나절 |
| anchor_meta + 커밋 매칭 로직 | 위 파일에 scoring 함수 | 1일 |
| CLI 명령 `vib plan-progress` | `vibelign/commands/vib_plan_progress_cmd.py` (신규) | 반나절 |
| Tauri IPC + GUI 카드 | `planning_progress` Tauri command + `PlanProgressCard.tsx` | 1일 |
| watch 연동 자동 갱신 | `watch_engine.py`에 progress 재계산 트리거 | 반나절 |

**총 예상: 3~4일**

## 개선 2: 코알못 단순화

### 문제
현재 기획방은 페르소나 선택, 모드 선택(Instant/초안/전체), @멘션 등 코알못한테 불필요한 결정을 요구함. "뭘 만들고 싶은지"만 말하면 되는데 UI가 복잡.

### 해결
초보 모드에서는 "아이디어 입력 → 자동 진행 → 기획안 생성"까지 사용자 개입 최소화.

### 구체적 구현

**초보 모드 흐름:**

```
사용자: "예약 앱 만들고 싶어"
  │
  ├→ VibeLign이 자동으로 지오(검토자)에게 먼저 보냄
  │   └→ 지오: "어떤 예약이에요? 식당? 병원? 미용실?"
  │
  ├→ 사용자: "식당 예약"
  │   └→ 클로이(설계자) 자동 투입: "식당 예약이라면 이런 기능이 필요해요: ..."
  │
  ├→ 사용자가 "좋아" 또는 수정 요청
  │   └→ 미나(탐색자) 자동 투입: "비슷한 앱을 보면 이런 기능도 있어요: ..."
  │
  └→ 충분히 논의됐으면 자동으로 기획안 생성
      "기획안이 완성됐어요! [보기] [AI로 만들기]"
```

**사용자가 할 일:** 아이디어 입력 + 질문에 대답. 끝.
**사용자가 안 해도 되는 것:** 페르소나 선택, 모드 선택, @멘션.

**구현 방식:**

- `PlanningRoom.tsx`에 `beginnerMode` 상태 추가
- 초보 모드일 때:
  - `PlanningModeSelector` 숨김
  - 페르소나 버튼 숨김
  - 자동 페르소나 라우팅: 첫 메시지 → 지오, 2번째 → 클로이, 3번째 → 미나 (또는 대화 맥락에 따라 AI가 결정)
  - "기획안으로 저장" 버튼 대신 **자동 저장 제안** ("충분히 논의된 것 같아요. 기획안으로 정리할까요?")

- 고급 모드: 현재 그대로 유지 (페르소나 선택, 모드, 멘션 다 노출)

**토글 위치:**
- 설정에서 "초보 모드" 체크박스 (코알못 UX 기획안의 전체 토글과 연동)
- 또는 기획방 진입 시 "간단하게 / 직접 선택할게" 선택지

### 자동 기획안 생성 판단 기준

"충분히 논의됐는지"의 기준:
- 대화 턴 수 ≥ 6 (사용자 3 + 페르소나 3)
- "핵심 기능" 관련 키워드가 3개 이상 등장
- 사용자가 "좋아", "그래", "그걸로" 같은 긍정 응답을 한 경우

이 조건 만족하면 "기획안으로 정리할까요?" 프롬프트 자동 표시.

### 작업 범위

| 항목 | 위치 | 예상 |
|------|------|------|
| `beginnerMode` 상태 + 토글 | `PlanningRoom.tsx` | 반나절 |
| 자동 페르소나 라우팅 로직 | `PlanningRoom.tsx` 또는 새 `autoRouter.ts` | 1일 |
| UI 조건부 렌더링 (초보 모드에서 숨기기) | `PlanningRoom.tsx`, `PlanningModeSelector` | 반나절 |
| 자동 기획안 생성 제안 | `PlanningRoom.tsx` + 판단 로직 | 반나절 |

**총 예상: 2~3일**

## 개선 3: 지식 저장고 연동

### 문제
기획방 채팅에서 나온 중요 결정("왜 이 구조를 선택했는지", "어떤 대안을 검토했는지")이 채팅 로그에만 남아서 나중에 찾기 어려움. 기획안 md에는 최종 결론만 남고 논의 과정은 휘발됨.

### 해결
채팅 중 중요 결정을 자동/수동으로 `knowledge/decisions/`에 저장.

### 구체적 구현

**수동 저장 (즉시 구현 가능):**

채팅 메시지 옆에 "💡 지식 저장" 버튼 추가.

```
┌─────────────────────────────────┐
│ 🔵 지오: 로그인은 OAuth로       │
│ 하는 게 좋아요. 자체 인증은     │
│ 보안 유지보수가 부담이에요.     │
│                      [💡 저장]  │
└─────────────────────────────────┘
```

클릭하면:
```json
// knowledge/decisions/oauth-선택-이유.md 자동 생성
---
title: 로그인 인증 방식 — OAuth 선택
date: 2026-06-04
source: 기획방 채팅 (예약 앱)
persona: 지오
---

OAuth로 하는 게 좋다. 자체 인증은 보안 유지보수가 부담.

## 맥락
사용자가 "식당 예약 앱"을 기획 중, 로그인 방식 논의에서 결정됨.

## 대안 검토
- 자체 인증: 보안 유지보수 부담
- OAuth: 구글/카카오 로그인으로 빠르게 구현 가능
```

**자동 저장 (2차 구현):**

AI가 채팅을 분석해서 "결정이 내려진 순간"을 자동 감지:
- "~로 하자", "~가 낫겠다", "~로 결정" 같은 패턴
- 페르소나가 명시적 추천을 하고 사용자가 동의한 턴

자동 감지되면 "이 결정을 지식으로 저장할까요?" 프롬프트 표시.

**기획안 완성 시 일괄 추출 (3차 구현):**

기획안 md 저장 시점에 채팅 전체를 분석해서:
- 주요 결정 3~5개 자동 추출
- 각각 `knowledge/decisions/` 에 저장
- 기획안 md에 "관련 지식 문서" 링크 섹션 추가

### 지식 저장고 폴더 구조 (기획방 연동)

```
knowledge/
├── decisions/
│   ├── oauth-선택-이유.md          ← 기획방 채팅에서 저장됨
│   ├── 식당-카테고리-분류-방식.md  ← 기획방 채팅에서 저장됨
│   └── ...
└── plans-archive/
    └── 예약-앱-v1.md              ← 기획안 구현 완료 후 여기로 이동
```

### 작업 범위

| 항목 | 위치 | 예상 |
|------|------|------|
| "💡 지식 저장" 버튼 추가 | `PlanningRoom.tsx` 채팅 메시지 컴포넌트 | 반나절 |
| 저장 로직 (md 파일 생성) | Python `vibelign/core/knowledge_store.py` (신규) 또는 Rust | 반나절 |
| Tauri IPC `save_knowledge` | `lib.rs` + CLI `vib knowledge save` | 반나절 |
| knowledge 폴더 자동 생성 | `vib start`에 통합 | 30분 |
| 자동 감지 (2차) | 결정 패턴 매칭 로직 | 1일 |
| 기획안 완성 시 일괄 추출 (3차) | AI 호출 또는 규칙 기반 | 1~2일 |

**1차 수동 저장: 1~2일**
**2차 자동 감지: +1일**
**3차 일괄 추출: +1~2일**

## 작업 순서 (권장)

### Phase 1 — 기획 진행률 대시보드 (3~4일)
- 사용자 체감이 가장 큼
- "기획한 거 얼마나 됐어?"에 자동으로 답해주는 기능

- 1차로 수동 "💡 지식 저장" 버튼만 먼저
- 2차, 3차는 사용 패턴 보고 결정

**전체 예상: 7~12일**

## 성공 지표

### 기획 진행률 대시보드
- 사용자가 "어디까지 했어?" 수동 질문 빈도 감소
- 기획안 대비 구현 완료율 추적 가능
- 미구현 항목 식별 → 다음 작업 자동 제안

### 코알못 단순화
- 기획방 첫 사용자의 기획안 완성률 (현재 vs 개선 후)
- 기획방 진입 → 기획안 생성까지 평균 소요 시간
- 페르소나 선택에서 혼란 이탈하는 비율 감소

### 지식 저장고 연동
- 기획방 채팅에서 저장된 지식 문서 수 (월평균)
- 저장된 지식이 나중에 참조된 횟수
- "왜 이렇게 만들었지?" 질문에 지식 문서로 바로 답변 가능한 비율

## 세 기능의 시너지

```
기획방 채팅
  ├→ [개선 2] 코알못도 쉽게 기획 완성
  ├→ [개선 3] 중요 결정을 지식으로 저장
  └→ 기획안 md 생성
       ├→ [개선 1] 진행률 자동 추적
       ├→ anchor_meta와 매칭
       └→ 구현 완료 시 knowledge/plans-archive/로 이동
```

세 기능이 합쳐지면 **"기획 → 구현 → 추적 → 회고"** 전체 사이클이 VibeLign 안에서 완성됨. 기획방이 단순 "문서 생성 도구"에서 **"프로젝트 라이프사이클 관리자"**로 진화.

## 개선 4: 기획 → 구현 자동 연결 (복사·붙여넣기 제거)

### 문제
기획방에서 기획안을 만든 후, 사용자가 기획안을 복사해서 Claude Code나 Cursor에 붙여넣어 구현을 시작해야 한다. 이 수동 단계가 흐름을 끊고 코알못에게 장벽이 됨.

### 해결: 4단계 점진적 자동화

#### 4-1. "AI로 만들기" 버튼 → 내장 터미널 자동 실행 (Phase 1)

기획안 완성 후 "이거 만들어줘" 버튼 한 번 → VibeLign 내장 터미널에서 Claude Code 자동 실행 + 기획안을 첫 프롬프트로 자동 입력.

```
기획방: 기획안 완성
  → [AI로 만들기] 버튼 클릭
  → 내장 터미널 자동 열림
  → Claude Code 자동 실행
  → 기획안 내용이 첫 프롬프트로 자동 입력
  → AI가 구현 시작
```

사용자: 버튼 한 번. 복사·붙여넣기 0.

**구현:**
- 내장 터미널 기획안(별도)과 통합
- 기획안 md를 읽어서 프롬프트 형태로 가공
- PTY stdin으로 자동 입력

**예상: 내장 터미널 구현 후 +1일**

---

#### 4-2. MCP로 자동 전달 (Phase 2)

VibeLign MCP 서버에 기획안 조회 도구 추가. AI 에이전트가 작업 시작할 때 VibeLign에서 기획안을 자동으로 읽어감.

**MCP 도구 추가:**

```python
# vibelign/mcp/mcp_planning_handlers.py (신규)

@tool("planning_get_current")
def planning_get_current():
    """현재 활성 기획안의 내용과 핵심 기능 목록을 반환"""
    return {
        "plan_file": "plans/예약-앱.md",
        "idea": "식당 예약 앱",
        "features": ["로그인", "예약 생성", "예약 목록", "알림", "관리자"],
        "full_content": "...(md 전문)...",
        "structure_plan": {...},  # vib plan-structure 결과 포함
        "progress": {"completed": 2, "total": 5}
    }

@tool("planning_get_next_task")
def planning_get_next_task():
    """기획안에서 아직 구현 안 된 다음 기능을 반환"""
    return {
        "feature": "알림 발송",
        "suggested_files": ["vibelign/core/notification.py"],
        "suggested_anchor": "NOTIFICATION_SEND",
        "context": "예약 확인 시 사용자에게 알림 발송"
    }
```

**사용 시나리오:**
```
사용자: "아까 기획한 거 만들어줘"
  → AI가 planning_get_current 호출 → 기획안 자동 수신
  → AI가 planning_get_next_task 호출 → 다음 할 일 확인
  → 구현 시작
```

복사·붙여넣기 없이, 사용자는 한 마디면 됨. 어떤 AI 도구에서든 동작.

**수정 위치:**
- `vibelign/mcp/mcp_planning_handlers.py` (신규)
- `vibelign/mcp/mcp_server.py`에 핸들러 등록
- `vib start`에서 MCP 등록 시 포함

**예상: 1~2일**

#### 4-3. 기획안 → 구현 지시서 자동 변환 (Phase 3)

기획안의 "핵심 기능"을 AI가 바로 구현할 수 있는 구체적 지시서로 변환.

**변환 흐름:**
```
기획안 (사용자용, 한국어):
  "식당 예약 앱. 로그인, 예약 생성, 목록 보기..."

  ↓ structure_planner + anchor_meta 연동

구현 지시서 (AI용):
  "1. vibelign/core/auth.py 생성 → LOGIN_HANDLER 앵커
      - OAuth 구현 (Google/Kakao)
      - 세션 관리 포함
   2. vibelign/core/reservation.py 생성 → RESERVATION_CREATE 앵커
      - 날짜/시간/인원 입력
      - 중복 예약 방지
   3. vibelign-gui/src/pages/ReservationList.tsx 생성
      - 예약 목록 테이블
      - 상태별 필터"
```

기획안의 추상적 기능 → 구체적 파일 경로·앵커명·구현 포인트까지 자동 생성. AI는 이걸 받으면 훨씬 정확하게 구현 가능.

**구현:**
- `vibelign/core/planning_to_spec.py` (신규)
- `structure_planner` + `project_map` 활용해서 파일 위치 결정
- `anchor_meta`의 기존 패턴 분석해서 앵커명 자동 제안
- 출력: 구현 지시서 md 또는 JSON

**예상: 2~3일**

#### 4-4. 단계별 구현 관리 (Phase 4 — 궁극)

기획안의 기능을 한 번에 다 던지지 않고, 하나씩 순서대로 AI에게 넘기고 각 단계를 검증.

**흐름:**
```
기획안: 기능 5개
  │
  ├→ 1단계: "로그인 기능"
  │   → AI 구현 → guard 체크 → doctor 체크
  │   → ✅ 통과 → 진행률 20% → checkpoint 생성
  │
  ├→ 2단계: "예약 생성"
  │   → AI 구현 → guard 체크 → ❌ 실패
  │   → VibeLign: "guard에서 문제 발견. 수정 중..."
  │   → AI 재시도 → ✅ 통과 → 진행률 40%
  │
  ├→ 3단계: "예약 목록"
  │   → ...
  │
  └→ 5단계 완료
      → 진행률 100%
      → "기획안 전체 구현 완료!"
      → 기획안 → knowledge/plans-archive/ 이동
```

**사용자 경험:**
- "만들어줘" 한 번 → 기다림
- 중간에 문제 생기면: "3단계에서 에러. 어떻게 할까요? [다시 시도] [건너뛰기] [직접 수정]"
- 완료 시: "5개 기능 전부 구현됐어요! guard 통과, doctor 이상 없음"

**이게 VibeLign이 에이전트가 되는 실체:**

```
perceive: 기획안 읽기 + 현재 코드 상태 파악
decide: 다음 구현할 기능 선택
act: AI에게 구현 지시 + 결과 수신
verify: guard + doctor 검증
revise: 실패 시 재지시 또는 사용자에게 문의
```

완전한 에이전트 루프.

**구현:**
- `vibelign/core/implementation_orchestrator.py` (신규)
- 기획안 파싱 → 단계 큐 생성
- 각 단계: MCP로 AI에게 지시 → 결과 수신 → guard/doctor 실행 → 결과에 따라 다음 단계 or 재시도
- 진행률 대시보드(개선 1)와 실시간 연동
- checkpoint 자동 생성 (각 단계 완료 시)

**예상: 2~3주 (가장 큰 작업)**

## 전체 로드맵 (개선 1~4 통합)

```
Phase 1 (1~2주)
  ├→ 개선 1: 기획 진행률 대시보드
  ├→ 개선 3-1차: 지식 저장 수동 버튼
  └→ 개선 4-1: "AI로 만들기" 버튼 (내장 터미널 연동)

Phase 2 (2~3주)
  ├→ 개선 2: 코알못 단순화 (초보 모드)
  ├→ 개선 4-2: MCP 자동 전달 (planning_get_current)
  └→ 개선 4-3: 기획안 → 구현 지시서 변환

Phase 3 (3~4주)
  ├→ 개선 3-2,3차: 지식 자동 감지 + 일괄 추출
  └→ 개선 4-4: 단계별 구현 관리 (에이전트 루프)
```

Phase 3 완료 시 VibeLign의 기획방은:
**"아이디어 한 줄 → 기획 → 구현 → 검증 → 추적 → 회고"**
전체 사이클을 한 곳에서 처리하는 **프로젝트 매니저 에이전트**.
