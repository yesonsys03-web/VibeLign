# PR 1 Spec: Onboarding Input Bar

작성일: 2026-06-02
브랜치: `feat/vibelign-product-renew`

## 목적

VibeLign의 첫 화면을 기능 카드 런처가 아니라 ChatGPT식 단일 입력 화면으로 바꾼다.

이 PR은 제품 인상 전환만 담당한다. `vib start` 자동 실행, 기획방 전환, LLM 호출, Home 재구성은 후속 PR에서 처리한다.

## 상위 결정

구현 순서는 다음을 진실 기준으로 삼는다.

```text
온보딩 화면 수정
→ plan 기능
→ 기존 Home 화면 정리
→ patch / CodeSpeak / plan-structure legacy 숨김과 제거 준비
```

PR 1은 첫 번째 항목만 구현한다.

## 사용자 목표

처음 실행한 사용자는 첫 화면에서 다음만 이해하면 된다.

1. 만들고 싶은 것을 입력한다.
2. 프로젝트 폴더를 고를 수 있다.
3. Claude Code도 같이 준비할지 선택할 수 있다.
4. 더 복잡한 기능은 고급 설정 뒤에 있다.

## 화면 구조

첫 화면의 중심은 단일 입력 바다.

```text
내 프로젝트를 안전하게 만들기

[ + ]  무엇을 만들고 싶나요?                         [ Instant v ] [ ● ]

[x] Claude Code도 자동으로 준비하기
고급 설정 보기
```

### 유지할 요소

- 작은 제목: `내 프로젝트를 안전하게 만들기`
- 입력 바
- 왼쪽 `+` 폴더 선택 버튼
- placeholder: `무엇을 만들고 싶나요?`
- 오른쪽 모드 표시: `Instant` — 이 드롭다운이 "쓰는 AI 선택"을 겸한다 (통합기획안 §4-1-2). PR 1에서는 UI 표시만, 실제 LLM 연결은 후속 PR(4/5)
- 오른쪽 전송 버튼: 검은 원 또는 동등한 icon button
- Claude Code 자동 준비 체크박스
- `고급 설정 보기` 토글
- `vib` 미설치, Git 미설치, Xcode CLT 미설치 같은 필수 시스템 경고
- 최근 프로젝트 목록은 고급 설정 내부 또는 입력 바 아래 secondary 영역으로 강등

### 제거할 요소

첫 화면에서 다음은 렌더링하지 않는다.

- `FEATURE_CARDS`
- `TERMINAL_LINES_DEFAULT` 터미널 시뮬레이터
- 도움말 질문하기 카드
- GitHub / Threads 카드
- `ONBOARDING_GUIDE_STEPS` 아코디언과 페이지 도트
- `patch`, `CodeSpeak`, `plan-structure`를 추천하는 문구
- `anchor`, `watch`, `guard`, `MCP`, `rules` 같은 내부 용어의 기본 노출

## PR 1 범위

### 포함

- `Onboarding.tsx` 첫 화면 레이아웃을 입력 바 중심으로 재구성
- 기존 첫 화면 카드와 터미널 시뮬레이터 렌더링 제거
- 도움말, GitHub 카드, 첫걸음 아코디언 렌더링 제거
- Claude Code 체크박스를 입력 바 아래 작은 보조 라인으로 배치
- 폴더 선택 버튼은 기존 폴더 선택 동작을 호출
- 텍스트 입력 상태를 보관하되 후속 기획방 전환은 하지 않음
- 전송 버튼은 PR 1에서는 안전한 비동작 또는 기존 완료 동작으로 연결하지 않는 명확한 placeholder 상태를 둠

### 제외

- `vib start` 자동 실행
- 5단계 progress streaming
- 입력 텍스트를 기획방으로 전달
- `plans/*.md` 생성
- LLM CLI 호출
- Home 3카드 재구성
- `vib patch`, CodeSpeak, `plan-structure` CLI 삭제
- watch / anchor / guard 자동화 변경

| 상황 | PR 1 동작 | 후속 PR |

| 텍스트만 입력하고 전송 | 폴더 선택이 필요하다는 짧은 안내를 같은 화면에 표시 | PR 3에서 기획방 질문으로 전환 |
| 폴더만 선택 | 선택한 폴더명을 입력 바 주변에 작게 표시 | PR 2에서 자동 start 실행 |
| 텍스트 입력 후 폴더 선택 | 텍스트를 유지하고 선택한 폴더를 표시 | PR 2/3에서 start + 기획방 전환 |
| 폴더 선택 후 텍스트 입력 | 폴더 표시를 유지하고 텍스트 입력 가능 | PR 2/3에서 start + 기획방 전환 |
| 텍스트도 폴더도 없음 | 전송 버튼 비활성 또는 안내만 표시 | 동일 |
| 폴더 선택 취소 | 기존 상태를 유지하고 오류로 보지 않음 | 동일 |
| 긴 텍스트 입력 | 입력 바가 깨지지 않고 가로 폭 안에서 스크롤 또는 확장 | PR 3에서 multiline 여부 결정 |
| 한글/영문/이모지 입력 | 입력값이 깨지지 않음 | 동일 |

## 상태와 에러

### `vib` 미설치

- 입력 바는 보여준다.
- 시스템 경고는 입력 바보다 작게 표시한다.
- 사용자가 먼저 설치 카드에 끌려가지 않게 한다.

### Git 미설치

- Git 경고는 유지할 수 있다.
- 첫 화면 중심을 밀어내지 않는다.

### Xcode CLT 미설치

- macOS에서 필요 경고는 유지할 수 있다.
- 고급 로그 형태로 커지지 않게 한다.

### Claude Code 상태

- 이미 준비됨: `Claude Code 준비됨`처럼 작은 상태로 표시
- 준비 필요: 체크박스 ON 권장
- 실패 상태: PR 1에서는 설치 실행을 하지 않으므로 과거 snapshot의 실패를 크게 노출하지 않음

## 고급 설정

`고급 설정 보기`를 눌렀을 때만 다음을 보여준다.

- 최근 프로젝트
- 시스템 상태 상세
- 설치/로그인 관련 상세 행동
- 개발자용 로그 보기

고급 설정 안에서도 `patch`, `CodeSpeak`, `plan-structure`를 첫 추천 흐름으로 보여주지 않는다.

## 금지어 계약

초보 첫 화면 기본 상태에는 다음 텍스트가 나타나면 실패다.

- `patch`
- `CodeSpeak`
- `plan-structure`
- `anchor`
- `watch`
- `guard`
- `MCP`
- `rules`
- `vib start`

예외:

- 고급 설정을 사용자가 펼친 뒤의 개발자 상세 영역
- 테스트 fixture 내부 문자열
- 사용자가 입력 바에 직접 친 텍스트. 금지어 계약은 VibeLign이 렌더하는 정적 문구에만 적용된다 (사용자가 "버그 패치 앱"처럼 자연어로 금지어를 입력하는 것은 위반이 아니다 — 금지어 테스트가 사용자 입력값을 검사 대상으로 삼지 않는다)
- 접근성 label 중 사용자가 시각적으로 보지 않는 내부 label. 단, screen reader 사용자를 고려해 label도 쉬운 말로 바꾸는 것을 권장한다.

## 구현 메모

현재 `vibelign-gui/src/pages/Onboarding.tsx`에는 다음 책임이 한 파일에 같이 있다.

- 첫 화면 기능 카드
- 터미널 시뮬레이터
- 도움말 카드
- GitHub/Threads 카드
- 첫걸음 아코디언
- 설치/검증 snapshot
- onboarding progress/log polling
- 최근 프로젝트 요약

PR 1에서는 전체 파일을 대규모 재작성하지 않는다. 렌더링 표면을 입력 바 중심으로 줄이고, 필요하면 새 작은 컴포넌트를 추출한다.

권장 컴포넌트 후보:

- `OnboardingPromptBar`
- `OnboardingAdvancedPanel`
- `ClaudeCodePrepOption`

## 자동 테스트

테스트 파일 후보:

```text
vibelign-gui/src/pages/__tests__/Onboarding.input-bar.test.tsx
```

필수 테스트:

1. 기본 렌더링
   - `무엇을 만들고 싶나요?`가 보인다.
   - `내 프로젝트를 안전하게 만들기`가 보인다.
   - Claude Code 체크박스가 보인다.

2. 제거된 첫 화면 요소
   - `코드맵 생성`이 보이지 않는다.
   - `AI 폭주 방지`가 보이지 않는다.
   - `원클릭 복구`가 보이지 않는다.
   - `AI 이동 자유`가 보이지 않는다.
   - `vibelign start`가 보이지 않는다.
   - GitHub/Threads 카드 문구가 보이지 않는다.

3. 금지어 계약
   - 기본 첫 화면에서 `patch`, `CodeSpeak`, `plan-structure`, `anchor`, `watch`, `guard`, `MCP`, `rules`가 보이지 않는다.

4. 폴더 선택 취소
   - dialog가 `null`을 반환해도 오류 UI가 뜨지 않는다.
   - 기존 텍스트 입력값은 유지된다.

5. 텍스트 입력
   - 한글 입력이 상태에 유지된다.
   - 긴 입력이 버튼과 겹치지 않는다.

6. 고급 설정
   - 기본 상태에서는 최근 프로젝트가 먼저 노출되지 않는다.
   - `고급 설정 보기` 후에만 secondary 정보가 표시된다.

## 수동 QA

### 시나리오 1: 완전 초보 첫 실행

```text
npm run dev 또는 Tauri dev 실행
→ Onboarding 첫 화면 열기
```

통과 기준:

- 입력 바가 첫 시선의 중심이다.
- 기능 카드/터미널/아코디언이 보이지 않는다.
- `patch`, `CodeSpeak`, `plan-structure`가 보이지 않는다.

### 시나리오 2: 텍스트 먼저 입력

```text
입력 바에 "예약 앱 만들고 싶어" 입력
→ 전송 버튼 클릭
```

통과 기준:

- 앱이 깨지지 않는다.
- 폴더 선택이 필요하다는 안내만 짧게 보인다.
- 기획방으로 이동하지 않는다.
- `vib start`가 실행되지 않는다.

### 시나리오 3: 폴더 선택 취소

```text
+ 버튼 클릭
→ 폴더 선택 dialog 취소
```

통과 기준:

- 오류로 처리하지 않는다.
- 입력 바가 유지된다.
- 기존 입력 텍스트가 사라지지 않는다.

### 시나리오 4: 고급 설정 열기

```text
고급 설정 보기 클릭
```

통과 기준:

- secondary 정보가 펼쳐진다.
- 첫 화면 기본 밀도는 낮게 유지된다.
- legacy 기능이 추천 경로처럼 보이지 않는다.

## 완료 정의

PR 1은 다음이 모두 만족되면 완료다.

1. 첫 화면의 중심이 단일 입력 바로 바뀌었다.
2. 기존 기능 카드, 터미널 시뮬레이터, 도움말 카드, GitHub 카드, 첫걸음 아코디언이 기본 렌더링에서 제거되었다.
3. Claude Code 자동 준비 체크박스가 입력 바 아래 보조 요소로 보인다.
4. 초보 첫 화면에서 금지어가 보이지 않는다.
5. 폴더 선택 취소, 텍스트만 입력, 빈 전송이 UI를 깨지 않는다.
6. PR 2 이후 작업과 혼동되지 않도록 자동 start, progress streaming, 기획방 전환, LLM 호출은 들어가지 않았다.
