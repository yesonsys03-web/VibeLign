# 기획방 폴백 사유 안내 — 구현계획 5

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`).

**Goal:** 페르소나의 1순위 모델이 폴백된 이유(로그인 필요/미설치/응답 실패)를 대화 메시지 배지에 함께 보여준다. "claude로 대체됨"만 떠서 "왜?"라고 헷갈리던 문제 해결.

**배경(실증):** 지오=codex 인데 codex 인증 토큰 만료(401 `token_expired`)로 `codex exec` 가 exit 1 + 빈 출력 → GUI 가 claude 로 자동 폴백 → "claude로 대체됨" 배지. 동작은 맞지만 **이유가 안 보여** 사용자가 혼란. Rust 대화 경로는 실패 사유를 구분하지 않고(현재 stderr 를 버림) 무조건 폴백한다. Python 계획서 경로의 `classify_cli_output`(로그인 인지)을 대화 경로에도 최소한 미러한다.

**설계 결정:** 폴백은 **유지**(사용자가 막히지 않게 다른 모델이 답함). 다만 **1순위가 왜 실패했는지** 사유를 배지에 표기.

**Architecture:** `run_persona_response` 가 시도 루프에서 **preferred(1순위) provider 의 실패 사유**를 분류·기록 → `PersonaRun.fallback_reason` → `PlanningChatMessage.fallback_reason` → TS 타입 → `PlanningMessages` 배지에 사유 라벨 추가.

**Tech:** Rust(cargo test), React/TS(Vitest). 구현계획 3(배지)·4(역할) 위에서 확장.

## 사유 분류

| reason | 트리거 | 배지 라벨 |
|---|---|---|
| `not_logged_in` | stdout/stderr 에 로그인/401/token expired/sign in 패턴 | 로그인 필요 |
| `not_installed` | 실행파일 PATH 에 없음 | 미설치 |
| `error` | 비-로그인 실패(비정상 종료·빈 출력·실행 오류) | 응답 실패 |

- 사유는 **preferred provider(=페르소나 namesake, order[0])** 의 실패에 대해서만 기록.
- 폴백이 안 일어나면(1순위 성공) `provider_used`·`fallback_reason` 모두 None → 배지 없음.

```rust
/// 실패 출력에서 로그인 문제를 감지. 아니면 "error".
fn classify_failure(stdout: &str, stderr: &str) -> String {
    let combined = format!("{stdout}\n{stderr}").to_lowercase();
    const LOGIN_PATTERNS: &[&str] = &[
        "not logged in", "not signed in", "please login", "please log in",
        "log out and sign in", "sign in again", "login required",
        "authentication required", "auth required", "unauthorized",
        "token_expired", "token expired", "401",
    ];
    if LOGIN_PATTERNS.iter().any(|p| combined.contains(p)) {
        "not_logged_in".to_string()
    } else {
        "error".to_string()
    }
}
```

```rust
    let preferred = order.first().cloned().unwrap_or_default();
    let mut preferred_reason: Option<String> = None;
    for provider in order {
        let is_preferred = provider == preferred;
        let Some((executable_name, args)) = provider_spec(&provider) else {
            if is_preferred { preferred_reason = Some("not_installed".to_string()); }
            continue;
        };
        let Some(executable) = find_executable(executable_name) else {
            if is_preferred { preferred_reason = Some("not_installed".to_string()); }
            continue;
        };
        let mut cmd = Command::new(executable);
        cmd.args(args);
        cmd.arg(&prompt);
        cmd.current_dir(project_dir);
        cmd.stdin(Stdio::null());
        cmd.env("PATH", augmented_vib_path());
        cmd.env("NO_COLOR", "1");
        hide_console(&mut cmd);
        match cmd.output() {
            Ok(output) if output.status.success() => {
                let content = String::from_utf8_lossy(&output.stdout).trim().to_string();
                if !content.is_empty() {
                    return PersonaRun {
                        content,
                        status: "ok".to_string(),
                        provider_used: fallback_provider_used(&preferred, &provider),
                        fallback_reason: if is_preferred { None } else { preferred_reason.clone() },
                    };
                }
                if is_preferred { preferred_reason = Some("error".to_string()); }
            }
            Ok(output) => {
                if is_preferred {
                    let stdout = String::from_utf8_lossy(&output.stdout);
                    let stderr = String::from_utf8_lossy(&output.stderr);
                    preferred_reason = Some(classify_failure(&stdout, &stderr));
                }
            }
            Err(_) => {
                if is_preferred { preferred_reason = Some("error".to_string()); }
            }
        }
    }
```

```tsx
const FALLBACK_REASON_LABEL: Record<string, string> = {
  not_logged_in: "로그인 필요",
  not_installed: "미설치",
  error: "응답 실패",
};

// === ANCHOR: PLANNINGMESSAGES_FALLBACKBADGELABEL_START ===
export function fallbackBadgeLabel(message: PlanningChatMessage): string | null {
  if (!message.providerUsed) return null;
  const base = `${message.providerUsed}로 대체됨`;
  const reason = message.fallbackReason ? FALLBACK_REASON_LABEL[message.fallbackReason] : undefined;
  return reason ? `${base} · ${reason}` : base;
}
// === ANCHOR: PLANNINGMESSAGES_FALLBACKBADGELABEL_END ===
```
