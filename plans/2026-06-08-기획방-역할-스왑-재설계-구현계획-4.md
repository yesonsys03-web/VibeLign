# 기획방 역할 스왑 재설계 — 구현계획 4

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** 페르소나(이름+모델)는 고정 덩어리로 두고, **역할(설계/검토/탐색/조교)을 사용자가 1:1 맞교환**으로 재배정하게 한다. 모델은 namesake 고정 + 자동 폴백. 이는 구현계획 1·2 의 "provider 수동 재배정"을 **역할 재배정**으로 교체하는 재작업이다.

**왜:** 페르소나 이름이 모델 작명(클로이=Claude, 딥시기=DeepSeek …)이라, 모델을 바꾸면 이름↔모델이 어긋난다. 바뀌어야 하는 건 **역할**이다.

**Architecture:** 역할을 페르소나에서 분리한 레지스트리(설계/검토/탐색/조교 = 라벨+프롬프트 설명)를 Rust·Python 양쪽에 둔다. 전역 설정 `planning_personas.personas[id] = { enabled, role }` 가 페르소나↔역할 배정을 담는다(provider 필드 제거). 모델은 항상 페르소나 namesake + 폴백. 프롬프트는 "페르소나 이름 + 배정된 역할 설명"으로 생성. Settings 는 역할 드롭다운 + 1:1 스왑.

**Tech:** Rust(cargo test), Python(pytest), React/TS(Vitest). 구현계획 1·2·3 위에서 수정.

**관련:** `...-design.md`, 구현계획 1·2·3.

## 역할 / 페르소나 모델 (양쪽 공통)

역할 레지스트리:
| role id | 라벨 | 프롬프트 역할 | 비고 |
|---|---|---|---|
| `design` | 설계 | 제품 설계자. 막연한 아이디어를 기능 흐름과 화면 구조로 구체화. | 설계 습관 nudge 보유 |
| `review` | 검토 | 기획 검토자. 빠진 조건·위험한 가정·정해야 할 결정을 짚음. | |
| `explore` | 탐색 | 사용자 탐색자. 실제 사용 상황·입력 방식·엣지케이스를 질문. | |
| `assist` | 조교 | 조교. 다른 역할의 설명·결정을 쉽게 풀고 사용자 질문에 답함. | |

페르소나(이름+모델 고정, 기본 역할):
| persona | 이름 | 모델(adapter) | 기본 role |
|---|---|---|---|
| chloe | 클로이 | claude | design |
| gio | 지오 | codex | review |
| mina | 미나 | agy | explore |
| deepseek | 딥시기 | opencode | assist |

설정 스키마(전역 `~/.vibelign/gui_config.json`):
```jsonc
"planning_personas": {
  "version": 1,
  "personas": {
    "chloe":    { "enabled": true, "role": "design" },
    "gio":      { "enabled": true, "role": "review" },
    "mina":     { "enabled": true, "role": "explore" },
    "deepseek": { "enabled": true, "role": "assist" }
  }
}
```
- `role` 미지정/미지값 → 페르소나 기본 role.
- `provider` 필드는 **제거**(더 이상 읽지 않음). 모델은 항상 namesake + 폴백.
- 1:1 불변식(각 역할 정확히 한 명)은 **프런트가 스왑으로 보장**. 백엔드는 어떤 값이 와도 안전하게 동작(중복이면 그대로 두 명이 같은 역할 — 크래시 없음).

## 범위

- **포함:** 역할 레지스트리, 설정 provider→role 전환, 모델 namesake 고정+폴백, 프롬프트가 배정 역할 사용, Settings 역할 드롭다운 1:1 스왑. 대화 경로(Rust) + 계획서 경로(Python) 둘 다.
- **딥시기/조교:** 대화 경로는 4명 4역할 1:1. 계획서 경로는 딥시기 미포함(3명) — 실행되는 페르소나에 한해 배정 역할을 따른다. `assist` 역할이 계획서 경로 페르소나에 배정되면 그 역할 프롬프트로 동작(허용, 크래시 없음).
- **유지:** ON/OFF, 자동 폴백, "대체됨" 배지(구현계획 3 — 모델이 namesake 고정이라 폴백/배지 그대로 유효).
- **제외:** 기획방 인라인 편집(여전히 보류).

from __future__ import annotations

from dataclasses import dataclass
from typing import Final


@dataclass(frozen=True, slots=True)
class PlanningRole:
    id: str
    label: str
    prompt_role: str


DESIGN_ROLE: Final = PlanningRole("design", "설계", "설계자")
REVIEW_ROLE: Final = PlanningRole("review", "검토", "검토자")
EXPLORE_ROLE: Final = PlanningRole("explore", "탐색", "탐색자")
ASSIST_ROLE: Final = PlanningRole("assist", "조교", "조교")

ORDERED_ROLES: Final = (DESIGN_ROLE, REVIEW_ROLE, EXPLORE_ROLE, ASSIST_ROLE)
ORDERED_ROLE_IDS: Final = tuple(role.id for role in ORDERED_ROLES)
ROLES_BY_ID: Final = {role.id: role for role in ORDERED_ROLES}


def role_for_id(role_id: str) -> PlanningRole:
    role = ROLES_BY_ID.get(role_id)
    if role is None:
        raise ValueError(f"unsupported planning role: {role_id}")
    return role
# === ANCHOR: ROLES_END ===

```tsx
import { useEffect, useState } from "react";
import { getPlanningPersonas, setPlanningPersonas, probePlanningProviders, type PlanningPersonaConfigMap } from "../lib/vib";
import { PLANNING_PERSONAS } from "../pages/planning/PlanningPersonas";

export const PLANNING_ROLE_OPTIONS = [
  { id: "design", label: "설계" },
  { id: "review", label: "검토" },
  { id: "explore", label: "탐색" },
  { id: "assist", label: "조교" },
] as const;

// 페르소나 고정 모델(이름과 한 덩어리). UI 표시/설치확인용.
const PERSONA_MODEL: Record<string, string> = { chloe: "claude", gio: "codex", mina: "agy", deepseek: "opencode" };
const DEFAULT_ROLE: Record<string, string> = { chloe: "design", gio: "review", mina: "explore", deepseek: "assist" };

export interface EffectivePersona { enabled: boolean; role: string; }

export function effectivePersona(map: PlanningPersonaConfigMap, id: string): EffectivePersona {
  const entry = map[id] ?? {};
  return { enabled: entry.enabled ?? true, role: entry.role ?? DEFAULT_ROLE[id] ?? "design" };
}

/** persona 에 newRole 을 주고, 그 역할을 갖고 있던 persona 와 1:1 맞교환. */
export function applyRoleSwap(map: PlanningPersonaConfigMap, id: string, newRole: string): PlanningPersonaConfigMap {
  const current = effectivePersona(map, id);
  if (current.role === newRole) return map;
  const ids = PLANNING_PERSONAS.map((p) => p.id);
  const holder = ids.find((other) => other !== id && effectivePersona(map, other).role === newRole);
  const next: PlanningPersonaConfigMap = { ...map };
  next[id] = { ...effectivePersona(map, id), role: newRole };
  if (holder) {
    next[holder] = { ...effectivePersona(map, holder), role: current.role };
  }
  return next;
}

export function PlanningPersonaSettings() {
  const [map, setMap] = useState<PlanningPersonaConfigMap>({});
  const [installed, setInstalled] = useState<string[] | null>(null);
  const [saving, setSaving] = useState(false);

  useEffect(() => {
    getPlanningPersonas().then(setMap).catch(() => setMap({}));
    probePlanningProviders().then(setInstalled).catch(() => setInstalled([]));
  }, []);

  async function persist(next: PlanningPersonaConfigMap) {
    setMap(next); setSaving(true);
    try { await setPlanningPersonas(next); } finally { setSaving(false); }
  }

  return (
    <div className="card" style={{ marginTop: 16 }}>
      <div style={{ fontWeight: 700, fontSize: 12, marginBottom: 8, textTransform: "uppercase", letterSpacing: 1 }}>기획방 페르소나</div>
      <div style={{ fontSize: 11, color: "#777", lineHeight: 1.6, marginBottom: 12 }}>
        각 AI가 어떤 역할을 맡을지 고르세요. 역할을 바꾸면 그 역할을 갖고 있던 AI와 서로 맞바뀝니다.
        모델이 없거나 로그인이 안 되어 있으면 다른 모델로 자동 대체돼요. {saving && <span style={{ color: "#4DFF91" }}>저장 중…</span>}
      </div>
      {PLANNING_PERSONAS.map((persona) => {
        const eff = effectivePersona(map, persona.id);
        const model = PERSONA_MODEL[persona.id] ?? "";
        const modelOk = installed ? installed.includes(model) : true;
        return (
          <div key={persona.id} style={{ display: "flex", alignItems: "center", gap: 10, marginBottom: 10, flexWrap: "wrap" }}>
            <span style={{ fontWeight: 700, fontSize: 12, minWidth: 64 }}>{persona.label}</span>
            <span title={modelOk ? "설치됨" : "미설치 — 다른 모델로 대체"} style={{ fontSize: 11, color: modelOk ? "#2f6f46" : "#A14B00", minWidth: 72 }}>
              {model}{modelOk ? "" : " (미설치)"}
            </span>
            <select
              value={eff.role}
              onChange={(e) => void persist(applyRoleSwap(map, persona.id, e.target.value))}
              style={{ background: "#1A1A1A", color: "#7DFF6B", border: "1px solid #333", borderRadius: 4, padding: "3px 8px", fontSize: 11 }}
            >
              {PLANNING_ROLE_OPTIONS.map((r) => (<option key={r.id} value={r.id}>{r.label}</option>))}
            </select>
            <button type="button" className="btn btn-sm"
              onClick={() => void persist({ ...map, [persona.id]: { ...eff, enabled: !eff.enabled } })}
              style={{ background: eff.enabled ? "#4DFF91" : "#1A1A1A", color: eff.enabled ? "#1A1A1A" : "#888", border: "2px solid #1A1A1A", fontWeight: 700, minWidth: 52 }}>
              {eff.enabled ? "ON" : "OFF"}
            </button>
          </div>
        );
      })}
    </div>
  );
}
```
