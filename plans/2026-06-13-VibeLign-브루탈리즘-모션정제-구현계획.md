# VibeLign 브루탈리즘 모션 정제 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** brutalism.css에 모션 토큰을 추가하고, 흩어진 하드코딩 타이밍을 토큰으로 통일하고, 인터랙티브 공유 클래스의 호버/포커스 모션을 일관화하고, `prefers-reduced-motion` 전역 가드를 추가한다. **단일 파일 수정으로 앱 전체 전파.**

**Architecture:** `vibelign-gui/src/styles/brutalism.css` 한 파일만 수정. `:root`에 `--dur`/`--dur-pop`/`--ease` 추가 → `.btn`·`.card`의 하드코딩 전환을 토큰화(동작 보존) → `.nav-tab`·`.input-field`에 토큰 전환 + `focus-visible` 하드 아웃라인 추가 → 파일 끝에 전역 reduced-motion 가드. 컴포넌트 `.tsx`는 건드리지 않는다.

**Tech Stack:** 순수 CSS. 검증은 빌드(vite/tsc) + 앱 실행 육안 + grep 회귀(토큰 존재·기존 인터랙션 보존).

**검증 특성:** CSS라 단위테스트 부적합. 각 태스크의 "검증"은 **grep으로 변경 착지/회귀 확인** + 마지막에 빌드 + 앱 육안.

**범위 밖(design §7):** 스페이싱 스케일 토큰 · 컴포넌트 마감 일관성 · 등장 모션 확대.

## Task VR2: 기존 전환 토큰화 — .btn, .card (동작 보존 + 카드 exit 버그 수정)

**Files:** Modify `vibelign-gui/src/styles/brutalism.css` (`.btn` 95행, `.card` 131-141행)

현재 `.btn`은 `transition: transform 0.05s, box-shadow 0.05s;`(95행). `.card`는 transition이 `:hover` 안에 있어(140행) **마우스를 떼면 전환이 안 먹는 버그** — base로 옮긴다.

- [ ] **Step 1: `.btn` 전환 토큰화** — 95행 교체:

변경 전:
```css
  transition: transform 0.05s, box-shadow 0.05s;
```
변경 후:
```css
  transition: transform var(--dur) var(--ease), box-shadow var(--dur) var(--ease);
```
(`.btn:hover`/`.btn:active`의 translate 값은 브루탈리즘 시그니처 — **그대로 유지**)

- [ ] **Step 2: `.card` 전환을 base로 이동 + 토큰화** — 131-141행 교체:

변경 전:
```css
.card {
  border: var(--border);
  box-shadow: var(--shadow);
  background: var(--white);
  padding: 16px;
}
.card:hover {
  transform: translate(-1px, -1px);
  box-shadow: var(--shadow-lg);
  transition: transform 0.1s, box-shadow 0.1s;
}
```
변경 후:
```css
.card {
  border: var(--border);
  box-shadow: var(--shadow);
  background: var(--white);
  padding: 16px;
  transition: transform var(--dur) var(--ease), box-shadow var(--dur) var(--ease);
}
.card:hover {
  transform: translate(-1px, -1px);
  box-shadow: var(--shadow-lg);
}
```

- [ ] **Step 3: 검증 (grep — 하드코딩 제거 + 토큰화 + 회귀)**

Run: `grep -nE "transform 0\.05s|transform 0\.1s|box-shadow 0\.1s" vibelign-gui/src/styles/brutalism.css`
Expected: **출력 없음**(하드코딩 타이밍 제거됨)

Run: `grep -nE "\.btn:active|\.card:hover" vibelign-gui/src/styles/brutalism.css`
Expected: 두 셀렉터 존재(시그니처 인터랙션 보존)

- [ ] **Step 4: 커밋**

```bash
git add vibelign-gui/src/styles/brutalism.css
git commit -m "feat(ui): .btn/.card 전환 토큰화 + .card exit 전환 버그 수정(base 이동)"
```

## Task VR3: .nav-tab / .input-field 전환 + focus-visible 하드 아웃라인

**Files:** Modify `vibelign-gui/src/styles/brutalism.css` (`.nav-tab` 277-295행, `.input-field` 358-369행)

`.nav-tab`은 전환 없음(bg 즉각 변경) + focus 표시 없음. `.input-field`는 `outline: none`(365행)이라 **포커스 링이 사라진 a11y 갭** — focus-visible 하드 아웃라인으로 보강.

- [ ] **Step 1: `.nav-tab` 전환 + focus-visible 추가**

`.nav-tab` 블록 마지막 속성(`flex-shrink: 0;`, 290행) **뒤, 닫는 `}` 앞**에 추가:
```css
  transition: background var(--dur) var(--ease);
```
그리고 `.nav-tab:hover { background: var(--gray); }`(292행) **뒤**에 새 규칙 추가:
```css
.nav-tab:focus-visible { outline: 3px solid var(--primary); outline-offset: -3px; }
```

- [ ] **Step 2: `.input-field` 전환 + focus-visible 추가**

`.input-field` 블록 마지막 속성(`color: var(--black);`, 367행) **뒤, 닫는 `}` 앞**에 추가:
```css
  transition: box-shadow var(--dur) var(--ease);
```
그리고 `.input-field:focus { box-shadow: var(--shadow); }`(369행) **뒤**에 새 규칙 추가:
```css
.input-field:focus-visible { outline: 3px solid var(--primary); outline-offset: 2px; }
```

- [ ] **Step 3: 검증 (grep)**

Run: `grep -nE "focus-visible" vibelign-gui/src/styles/brutalism.css`
Expected: 2줄(`.nav-tab:focus-visible`, `.input-field:focus-visible`)

Run: `grep -cE "var\(--dur\) var\(--ease\)" vibelign-gui/src/styles/brutalism.css`
Expected: 4 이상(.btn 2 + .card 1 + nav-tab 1 + input-field 1 = 토큰 전환 사용처)

- [ ] **Step 4: 커밋**

```bash
git add vibelign-gui/src/styles/brutalism.css
git commit -m "feat(ui): .nav-tab/.input-field 토큰 전환 + focus-visible 하드 아웃라인(a11y)"
```

## Task VR4: 전역 prefers-reduced-motion 가드

**Files:** Modify `vibelign-gui/src/styles/brutalism.css` (파일 끝 `/* === ANCHOR: BRUTALISM_CSS_END === */` 앞)

현재 `prefers-reduced-motion` 1곳뿐. 전역 가드 추가 → 마스코트/가이드 keyframe 포함 모든 모션 완화.

- [ ] **Step 1: 전역 가드 추가** — 파일 끝, END 앵커 주석 **앞**에 삽입(END 앵커가 없으면 파일 맨 끝):

```css
/* 동작 줄이기 선호 시 모든 모션 완화 (전역 가드) */
@media (prefers-reduced-motion: reduce) {
  *, ::before, ::after {
    animation-duration: .01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: .01ms !important;
  }
}
```

- [ ] **Step 2: 검증 (grep)**

Run: `grep -c "prefers-reduced-motion" vibelign-gui/src/styles/brutalism.css`
Expected: 2 이상(기존 1 + 신규 전역 1)

Run: `grep -nE "ANCHOR: BRUTALISM_CSS_END" vibelign-gui/src/styles/brutalism.css`
Expected: END 앵커가 가드 블록 **뒤**에 있음(앵커 경계 보존). END 앵커가 원래 없었으면 이 체크는 건너뜀.

- [ ] **Step 3: 커밋**

```bash
git add vibelign-gui/src/styles/brutalism.css
git commit -m "feat(ui): 전역 prefers-reduced-motion 가드(마스코트 포함 모션 완화)"
```
