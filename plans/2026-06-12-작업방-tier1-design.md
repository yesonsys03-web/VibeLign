# 작업방 (Work Room) Tier 1 — 설계 (design)

> 작성 2026-06-12 · 가이드 줄 UX 트라이얼 세션에서 사용자와 단계별 합의
> 근거: 표준 여정 4️⃣(AI에게 작업 시키기)는 사용자가 앱 밖으로 나가는 유일한 지점 = **최대 이탈 절벽**(가이드레이어 design §2, GuideStrip 주석). 사용자 문제 제기: "사용자가 바이브라인을 벗어나면 이걸 쓸 이유가 없어지잖아."
> 선행: 강PD 가이드레이어(2026-06-10) · 가이드 줄 시제 라벨링·점진적 노출(d9c9e17) · MCP 피벗 결정(2026-05-13, 숙련자 트랙)

## 0. 한 줄 정의

**기획방이 페르소나 CLI를 서브프로세스로 돌리듯, 작업방은 사용자의 코딩 CLI(Claude Code 등)를 헤드리스로 돌려 — 체크포인트→실행→guard 안전 시퀀스를 앱 안에서 완결한다.**

## 1. 확정 결정 (세션 합의 이력)

| 결정 | 선택 | 기각 대안 · 이유 |
|---|---|---|
| 통합 수준 | **Tier 1: 헤드리스 CLI 작업방** | Tier 0(내장 터미널) — 코드베이스 재사용 0(PTY·xterm 전무), 안전 배선 우회, PTY 유지보수 세금. Tier 1 후 재평가 |
| 실행 주체 | **BYO CLI** — 사용자가 설치·로그인한 CLI를 서브프로세스로 | 자체 키 내장/웹 UI 자동화 — 정책 위반. 헤드리스 모드는 각사 공식 지원 경로 |
| MVP 프로바이더 | **Claude Code, Codex부터** (코딩 에이전트 검증된 CLI) | 4종 동시(agy·opencode 포함) — 파일 수정 품질 미검증 CLI로 첫 경험 실패 시 가이드 여정 신뢰 붕괴 |
| 종료→검증 전환 | **프로세스 종료 감지 = 자동 guard** | "다 했어요" 수동 출구 유지 — 작업방 안에서 실행하면 종료가 신호로 잡히므로 수동 출구 불필요(외부 실행 폴백에는 유지) |
| 후속 확장 축 | **Tier 1 → `--resume` 멀티턴 → Tier 2(쓰기 전 guard)** | Tier 0 후속 — 멀티턴이 Tier 0 가치(대화형) 대부분을 신규 의존성 0으로 커버 |

## 2. 사용자 여정 변화 (4️⃣~6️⃣ 자동화)

| 단계 | 현재 (외부 도구) | 작업방 도입 후 |
|---|---|---|
| 3️⃣ 체크포인트 | 수동 — 백업 탭 가서 저장 | **자동** — "AI에게 시키기" 누르면 실행 전 자동 저장 |
| 4️⃣ AI 작업 | 지시문 복사 → 앱 이탈 → 외부 CLI 붙여넣기 | **앱 내** — 작업방에서 실행, 출력 실시간 스트리밍 |
| 4️⃣→5️⃣ 전환 | 신호 감지 불가 → "다 했어요" 수동 버튼 | **자동** — 프로세스 종료 감지 → guard 자동 실행 |
| 5️⃣ 검증 | 홈 가서 '상태 확인' 수동 클릭 | **자동** — guard 결과가 작업방에 바로 표시, 가이드 5️⃣/6️⃣ 자동 전환(기존 `inferStep` changed-set 신호 그대로 동작) |
| 6️⃣ 저장/되돌리기 | 백업 탭 | 작업방 결과 화면에서 원클릭 (성공→체크포인트 / 실패→되돌리기) |

가이드 스트립 4️⃣의 주행동은 "지시문 복사하러 가기" → **"작업방에서 시키기"**로 진화. 외부 도구 선호 사용자를 위해 "직접 붙여넣기(복사)" 보조 동선은 유지(이때만 "다 했어요" 수동 출구 노출).

## 3. 아키텍처 — 재사용 자산 4 + 신규 1

코드 기준 타당성 검토(2026-06-12 세션) 결과. 빌딩블록 5개 중 4개가 검증된 기성품.

| 능력 | 소스 | 재사용 방식 |
|---|---|---|
| CLI 탐지·실행 | `planning_persona.rs` — `INTERNAL_PROVIDER_PRIORITY`, Windows `.cmd/.exe` 셔임 해석, `augmented_vib_path()`, `classify_failure` | 그대로 import. 파이프 교착·Windows 좀비 리더 처리 이력 포함 |
| 스트리밍 → UI | `vib_bridge.rs:253` `run_vib_with_progress` — 라인 리더 스레드 → `app.emit(event_name)` → 프런트 `listen` | 패턴 복제 (vib 고정 제거가 신규 작업의 핵심) |
| 장기 실행 수명주기 | `watch.rs` — `Arc<Mutex<Runtime>>` child 등록, stop 명령, `Drop` kill, 종료 핸들 | 패턴 복제 → 취소 버튼·앱 종료 정리 |
| 지시문 생성 | `planningInstruction.ts` `buildPlanningWorkInstruction` + 페르소나→CLI 매핑(`PERSONA_CLI_DETAILS`) | 클립보드 대신 이 문자열을 헤드리스 입력으로 직접 전달. 기획 담당 페르소나 → 실행 CLI 자동 연결 |
| 체크포인트/guard 호출 | `vib_bridge.rs` `run_vib` | 실행 전후 자동 시퀀스에서 호출 |

**신규 1: 범용 CLI 스트리밍 러너** (Rust ~200–300줄)
- `run_vib_with_progress`의 `find_runtime_vib` 고정을 떼고 임의 실행 파일 + 인자 수용
- watch식 상태 등록(`WorkRoomState`) + `work_cancel` 커맨드
- 타임아웃: 페르소나용 120초 상한 폐기 → **무제한 + 취소 버튼 + idle 경고**(출력 무변화 N분 시 "멈춘 것 같아요" 배너)
- 이벤트 스키마: `work-output`(라인), `work-status`(running/done/cancelled/failed), `work-guard`(검증 결과)

### CLI 어댑터 (프로바이더별 헤드리스 차이 흡수)

| 프로바이더 | 호출 | 스트림 형식 | 권한 모드 |
|---|---|---|---|
| Claude Code | `claude -p <지시문> --output-format stream-json --mcp-config <vibelign 임시 설정>` | JSON 라인 | `--permission-mode acceptEdits` + vibelign MCP 주입(§9 확정) |
| Codex | `codex exec <지시문>` + MCP 서버 설정(`-c mcp_servers…`) | 텍스트/JSON | 샌드박스 기본값 + 워크스페이스 한정 + vibelign MCP 주입 |

어댑터 = `(실행파일, 인자 빌더, 스트림 파서, 권한 플래그)` 튜플. 기획방의 프로바이더별 인자 테이블과 같은 패턴. **위험 플래그(`--dangerously-skip-permissions` 류)는 절대 사용하지 않는다** — 안전이 제품 정체성.

## 4. 안전 시퀀스 (작업방의 존재 이유)

```
[AI에게 시키기]
  → vib checkpoint "작업방: <요청 요약>"   (자동, 실패 시 실행 중단 + 안내)
  → CLI 헤드리스 실행 (cwd=프로젝트, 스트리밍 표시, 취소 가능)
  → 종료 감지
  → vib guard --strict                    (자동)
  → 결과 화면: guard 요약 + 변경 파일 diff 링크
      ├─ 잘 됐어요 → [체크포인트 저장]
      ├─ 이어서 요청 (후속: --resume 멀티턴)
      └─ 잘 안 됐어요 → [되돌리기] (3️⃣ 시점 복귀)
```

Tier 0이 이 시퀀스를 우회(터미널 안 행위는 구조화 불가)하는 반면, Tier 1은 시퀀스를 **강제**할 수 있다 — 외부 도구 대비 본질적 우위.

## 5. UI 골격

- **위치(IA)**: 개발 단계 서브탭 `코드탐색 | 문서 | 작업방`. 진입은 ① 가이드 4️⃣ 주행동 ② 코드탐색의 지시문 영역(`PlanningInstructionActions`)에 "작업방에서 실행" 버튼
- **화면 구성** (위→아래): 지시문 카드(기획안 경로·요청 요약, 접기 가능) / CLI 선택(탐지된 것만 활성, 없으면 기존 "설치 도움받기" 분기 재사용) / 실행 버튼 / 스트리밍 출력 패널(`GuiCliOutputBlock` 재사용 후보) / 종료 후 guard 결과 + 행동 버튼
- **상태 표시**: 실행 중 가이드 스트립이 "4️⃣ 진행 중 — 작업방에서 실행 중" + 취소. 탭 이탈해도 백그라운드 계속(watch 패턴)

## 6. 정책·비용·신뢰

- **정책**: 사용자 본인 CLI의 공식 헤드리스 모드만 사용(BYO 계정). 자체 키 프록시·웹 UI 자동화 없음. 로그인 미탐지 시 실행 차단 + 안내(`probe_planning_providers` 재사용)
- **비용 투명성**: 실행 확인 다이얼로그에 "사용자 본인의 <CLI> 계정/요금제로 실행됩니다" 명시. stream-json의 토큰/비용 필드가 오면 결과 화면에 표시
- **신뢰**: 모든 출력 실시간 공개 + 실행 전 체크포인트 강제라 "최악의 경우 = 되돌리기 1번"

## 7. 리스크

| 리스크 | 대응 |
|---|---|
| 약한 CLI로 품질 붕괴 | MVP 프로바이더 2종 한정(§1). 확장은 품질 검증 후 |
| 장시간 실행/행 멈춤 | idle 경고 + 취소. 강제 타임아웃은 두지 않음(작업 길이 예측 불가) |
| 패키징 PATH 함정 | 기 해결(`augmented_vib_path`, 2026-06-07 페르소나 검증) — 작업방도 동일 경로 사용 |
| CLI 헤드리스 인터페이스 변경(업스트림) | 어댑터 격리 + 버전 탐지(`--version`) 후 미지원 안내 |
| checkpoint 실패(본 repo hash mismatch류) | 실행 전 체크포인트 실패 시 **실행 중단**이 기본(안전 우선). "그래도 실행" 우회는 경고와 함께 |

## 8. 마일스톤 (1인 기준 1~2주 MVP)

1. **러너** (2–3일): 범용 스트리밍 러너 + WorkRoomState + cancel + Claude Code 어댑터
2. **UI** (2–3일): 작업방 서브탭 + 스트리밍 패널 + 실행 전 확인 다이얼로그
3. **안전 배선** (1–2일): 자동 체크포인트→실행→guard 시퀀스 + 가이드 4️⃣ 주행동 교체 + 외부 실행 폴백 동선
4. **MCP 자동 주입** (1–2일, §9 확정으로 추가): 실행별 임시 MCP 설정 생성(사용자 전역 설정 불변) + 지시문에 "guard/anchor 도구 사용" 항목 추가 + 주입 실패 시 사후 guard만으로 폴백
5. **검증** (1–2일): Codex 어댑터 + Win/mac 패키징 빌드 실기기 + 취소/앱종료/idle 시나리오
6. **후속(별도 트랙)**: `--resume` 멀티턴 → Tier 2(쓰기 전 guard 가로채기)

## 9. 결정 완료 (2026-06-12 사용자 확정)

| 미결이었던 항목 | 확정 | 비고 |
|---|---|---|
| 권한 모드 수위 | **`acceptEdits` + vibelign MCP 자동 연결** | 권고(사후 guard만)보다 한 단계 높은 선택 — 실행 시 CLI에 vibelign MCP를 임시 주입(`claude -p --mcp-config` 류)해 guard/anchor primitive를 도구로 노출. 사용자 전역 설정은 건드리지 않는다. Tier 2의 선행 형태 |
| 실행 중 앱 종료 | **확인 후 중단** | 종료 확인 다이얼로그 → 자식 kill (watch 패턴). 고아 에이전트가 몰래 코드를 고치는 일 없음 |
| 자유 요청 허용 | **MVP는 기획안 기반만** | guard의 "약속 범위" 기준점 유지. 자유 입력은 멀티턴 도입 시 재검토 |
| 동시 실행 | **1개 제한** | worktree 격리 없는 다중은 체크포인트·guard 기준점 오염. 다중은 격리 인프라와 함께 |
