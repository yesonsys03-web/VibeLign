# 강 PD 가이드 레이어 — 설계 (design)

> 작성 2026-06-10 · 브레인스토밍 세션에서 사용자와 단계별 승인 완료
> 근거: 강 PD 피드백 본질(사용자 정정) = "초보자가 바이브라인에서 바이브코딩할 때 헷갈리지 않는 **순서와 로직** + 따라갈 수 있는 **상세한 가이드**". 상태 가시화·3단계 IA는 골격일 뿐, **"다음에 뭘 해야 하는지" 안내하는 레이어**가 미구현 갭.
> 선행: A안(9836462) 탭 한글화·유틸 분리·상태 스트립 / B안(df6c6f1..62694cd) 3단계 IA 셸 / 기획안 서브탭(2e9c473)

## 1. 확정 결정 (브레인스토밍 승인 이력)

| 결정 | 선택 |
|---|---|
| 가이드 깊이 | **행동 수준** — 각 단계 안의 구체적 다음 행동까지 안내 |
| 노출 방식 | **토글형** — 기본 켜짐, 설정/배너 ×로 끄기 가능 |
| 범위 | ① 상시 다음단계 안내 + ② 홈 허브 연동 + ③ 사용법 탭 재구성 (전부) |
| 상태 파악 | **접근법 A: 휴리스틱 추론 + 수동 보정** (명시적 저장 B, 정적 가이드 C 기각) |
| 배너 위치 | **A: 상단 스트립 통합** — 기존 상태 스트립(App.tsx 탭바 아래 얇은 줄)을 가이드 배너로 확장 (하단 바 B, 플로팅 카드 C 기각) |
| 홈 허브 연동 | **A: 카드 자체에 차례 표시** — "지금 할 차례" 카드 강조 + 카드 안 구체 행동 (진행 줄 B 기각) |
| 사용법 재구성 | **A: 따라하기 우선 + 명령어 사전은 아래로** — 한 페이지 아코디언 (서브탭 분리 B 기각) |

## 2. 표준 여정 (v3 — 확정)

가이드가 가르치는 단일 소스. 6단계:

| # | 단계 | 위치(IA) | 성격 | 안내 내용 |
|---|---|---|---|---|
| 1️⃣ | 프로젝트 열기 | 온보딩 | 자동 | 폴더 선택 → 코드맵·앵커 준비. **감시 자동 시작**(변경을 지켜보다 위반을 알려줌 — ⚠️ 자동 백업 아님). 가이드는 안심 메시지만 |
| 2️⃣ | 기획 | 기획 단계 · 기획방 | 행동 | "무엇을 만들고 싶은지" 적기 → AI 페르소나 대화 → 기획안 확정 |
| 3️⃣ | 체크포인트 저장 | 유지보수 · 백업 | **수동 행동** | AI에게 시키기 **전에** 현재 상태 저장 — "망쳐도 여기로 되돌아올 수 있게" |
| 4️⃣ | AI에게 작업 시키기 | 개발 · 코드탐색 | 행동(외부) | 기획안 기반 지시문 복사 → 외부 AI 도구(Claude Code 등)에 붙여넣기 |
| 5️⃣ | 결과 검증 | 유지보수 · 진단 | 행동 | guard 검사 — AI가 약속한 범위만 고쳤는지 확인 |
| 6️⃣ | 저장 또는 되돌리기 | 유지보수 · 백업 | 행동 | 잘 됐으면 체크포인트, 망쳤으면 undo(3️⃣ 시점 복귀) → 4️⃣로 반복 |

여정 사실 정정 이력(중요): 감시는 `Onboarding.tsx:122`에서 프로젝트 열기 시 자동 시작. 감시는 **백업이 아니라 변경 감시·위반 감지**(`watch_cmd.py` → watch_engine). 자동 백업 기능은 현재 없음 → 체크포인트는 수동 행동 단계로 유지.

## 3. 아키텍처

`stages.ts`(IA 단일 소스)와 같은 패턴. 백엔드(Rust/Python) 변경 0.

```
vibelign-gui/src/lib/nav/guide.ts                ← 신규, 순수 모듈 (vitest TDD)
vibelign-gui/src/components/nav/GuideStrip.tsx   ← 신규, 표시 컴포넌트 (기존 상태 스트립 대체·통합)
vibelign-gui/src/components/nav/StageHubCards.tsx ← 수정 (차례 표시)
vibelign-gui/src/pages/Home.tsx (manual_list 뷰)  ← 수정 (따라하기 아코디언 추가)
vibelign-gui/src/App.tsx                          ← 조립 + 신호 전달
```

### 3.1 `guide.ts` (순수)

- `JOURNEY_STEPS: JourneyStep[]` — 6단계 정의: `{ step: 1..6, label, shortAction, detailHowto, targetPage: Page, stage: Stage|null }`
- `inferStep(signals): 2|3|4|5|6` — 신호 → 현재 단계. **1️⃣은 반환하지 않음**: 프로젝트가 열려 있어야 가이드가 렌더되므로 1️⃣은 항상 완료 상태 (1️⃣은 `JOURNEY_STEPS` 정의에는 포함 — 사용법 따라하기 아코디언용). 입력:
  - `hasPlanDoc: boolean` (확정 기획안 존재 = `planningResult?.outputPath` 기준)
  - `hasCheckpoint: boolean`
  - `changedFileCount: number | null` (없으면 null = 신호 미배선)
  - `guardStatus: "ok" | "issue" | null`
  - `planningPending: boolean`
- 영속 상태 (localStorage):
  - `guideEnabled` — 전역 1개 (숙련도는 프로젝트와 무관)
  - `guideStepOverride:<projectDir>` — **프로젝트 경로별** (진행 위치가 프로젝트 간 새면 안 됨)

### 3.2 `GuideStrip.tsx` (표시만)

기존 App.tsx 상태 스트립(337~355행: 백업 유무 + 기획 진행)을 대체·통합:

```
🧭 지금: 3️⃣ 체크포인트 저장 — AI에게 시키기 전에 안전 저장  [백업으로 이동 →]  ‹ › ×   (우측: 기존 상태 신호 유지)
```

- `[이동 →]` 버튼 = 기존 `navigate(targetPage)` 재사용 (진단은 launch intent 경유 그대로)
- `‹ ›` = 수동 보정(override 기록), `×` = `guideEnabled=false` (설정에서 재활성)
- 가이드 off 시: 스트립은 기존 상태 표시(백업/기획 진행)만으로 축소 — 스트립 자체는 사라지지 않음

### 3.3 `StageHubCards.tsx` 연동

- 현재 단계가 속한 카드: 강조 테두리 + "지금 할 차례" 배지 + 구체 행동 1줄("→ 3️⃣ 체크포인트를 저장하세요")
- 완료 단계 카드: "✓ 완료" 톤다운 / 다음 카드: "다음" 표시
- 매핑: 기획 카드=2️⃣, 개발 카드=4️⃣, 유지보수 카드=3️⃣·5️⃣·6️⃣ (1:1 아님 — 카드는 "현재 단계 포함 여부"로 강조)

### 3.4 사용법 탭(따라하기 우선)

`Home`의 `manual_list` 뷰 상단에 여정 6단계 아코디언 추가, 기존 명령어 사전은 그 아래 유지:

- 각 단계 펼치면 초보자용 상세 하우투(①②③ 클릭 단위) + "지금 하러 가기 →"(navigate 재사용)
- 진입 시 **현재 단계(가이드 추론)가 자동으로 펼쳐진 상태**
- 신규 라우팅 없음 (한 페이지)

## 4. 엣지 케이스 · 규칙

1. **추론 충돌 시 뒤 단계 우선** (결정적 규칙). 예: 기획안 있음 + 코드 변경 감지 → 5️⃣.
2. **override 수명**: 추론 결과가 바뀌는 순간 자동 해제 — 실제 상황 변화가 사용자 고정보다 우선. 영구 어긋남 방지.
3. **기획 건너뛰는 사용자**: 2️⃣ 안내 강요 금지 — ‹ › 보정/× 끄기로 해소 (B안의 "선형 강제 금지" 원칙 유지).
4. **신호 로딩 전 배너 미렌더** — 잘못된 안내 깜빡임 방지.
5. **신호 미배선 시 보수적 생략**: `guardStatus`/`changedFileCount`가 없으면 해당 추론 규칙 생략(추측 안내 금지). 필요한 배선은 기존 Tauri 커맨드(git_status 등) 호출을 App으로 끌어올리는 수준 — 신규 백엔드 없음.

## 5. 검증 방침 (프로젝트 관례)

- `guide.ts` `inferStep`·override 수명 — **vitest TDD** (신호 조합별 케이스)
- 컴포넌트(GuideStrip·허브 카드·아코디언) — `tsc --noEmit` + tauri dev 육안 렌더 (CLAUDE.md "가장 작은 패치", UI 단위테스트 비요구)
- 회귀: 기존 `stages.test.ts` 통과, 기존 페이지 라우팅 무변경

## 6. 비목표 (명시)

- 자동 백업 기능 신설 (없음을 전제로 설계 — 생기면 3️⃣이 자동화될 수 있으나 별도 작업)
- 외부 AI 도구(4️⃣) 실행 감지 — 파일 변경으로 근사만
- 사용법 콘텐츠의 외부화(노션/전자책) — 펀딩 문서 §2.2 별도 트랙
- CS 단계 추가 (YAGNI 유지)

## 7. 구현 순서 제안 (계획서에서 상세화)

1. `guide.ts` + 테스트 (TDD, 순수)
2. `GuideStrip.tsx` — 기존 상태 스트립 대체·통합
3. `StageHubCards` 차례 표시
4. 사용법 따라하기 아코디언
5. 신호 배선 보강(guardStatus/changedFileCount — 필요 시)

각 단계 독립 커밋·출하 가능, 위험 오름차순.
