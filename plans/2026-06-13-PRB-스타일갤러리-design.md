## 0. 한 줄 정의

**디자인 프리뷰 갤러리를 5종으로 완성한다 — 나머지 4스타일(미니멀 SaaS·프루티거에어로·레트로다이너·리소그래프)에 시각 토큰+레시피+모션을 베이크하고, 모든 목업에 공용 컴포넌트 커버리지 체크리스트를 더해 더 풍성하게 한다.**

## 2. 데이터 — 나머지 4스타일 (`vibelign-gui/src/lib/design-preview/styles.ts`)

각 스타일 = 기존 `StyleSpec` 형태(tokens + recipe) + `motion`(모션 트랙에서 추가된 optional 필드). Rust는 `StyleSpec`을 통째 역직렬화하므로 **TS 데이터만 추가**(Rust 변경 없음).

토큰·시각 recipe는 PR A 구현계획 Task B1의 시작값 사용. 모션 시작값:

| 스타일 id | 모션 tokens(duration, easing) | 모션 recipe 요지 |
|---|---|---|
| `minimal-saas` | `200ms`, `cubic-bezier(.4,0,.2,1)` | 은은한 페이드/슬라이드·옅은 그림자 전환. 과하지 않게. reduced-motion서 정지 |
| `frutiger-aero` | `400ms`, `cubic-bezier(.34,1.56,.64,1)` | 플로팅·스프링 오버슈트(살짝 튕김)·광택 하이라이트 부드럽게. reduced-motion서 정지 |
| `retro-diner` | `300ms`, `cubic-bezier(.68,-.55,.27,1.55)` | 통통 바운스·간판 점멸 느낌의 강조. reduced-motion서 정지 |
| `risograph` | `120ms`, `steps(2,end)` 느낌 또는 짧은 선형 | 스냅·즉각·약간 거친 끊김. 부드러운 보간 지양. reduced-motion서 정지 |

> 모션 recipe는 자연어 prose(모션 트랙 §3.2 형식과 동일): CSS 모션 + 풀 어휘 함께, 프롬프트 빌더가 목업엔 "CSS만" 제약을 이미 덧붙임.

## 3. 공용 컴포넌트 커버리지 (`build_mockup_prompt`)

모든 목업 프롬프트에 무조건 추가되는 섹션(스타일·motion과 독립):
```
[컴포넌트 커버리지 — 기획안에 맞는 것을 충실히 표현]
내비게이션·버튼(주/보조)·카드·입력/폼·뱃지·탭·모달 또는 시트·리스트/테이블·빈 상태(empty state) 중
기획안 화면에 어울리는 것들을 실제 콘텐츠 예시로 포함하세요. 기획안과 무관한 컴포넌트는 넣지 마세요.
```
- 위치: 기존 `[규칙]` 블록 뒤(모션 섹션 앞 또는 뒤 — motion 섹션과 독립적이므로 순서 무관, 규칙 직후 권장).
- "맞는 것만/무관한 건 넣지 말 것" 가드 → 단순 화면에 불필요 컴포넌트 강제 방지.
- 효과: 5스타일 전부 + 기존 네오브루탈리즘 목업도 더 풍성해짐(의도).

## 4. 영향 범위 / 안전

- **Rust 변경 없음** (컴포넌트 커버리지 섹션 1개 제외): `build_mockup_prompt`에 무조건 섹션 추가. 4스타일은 TS 데이터.
- **하위호환**: motion optional(이미 안전 입증). 컴포넌트 커버리지는 공용이라 모든 스타일 균일 영향.
- **validator**: 컴포넌트가 많아져도 여전히 인라인 HTML/CSS — script·외부로드·핸들러 없음(현 validator 통과). 모션 동일(CSS).
- **기존 테스트 영향**: `build_mockup_prompt` 출력이 항상 컴포넌트 섹션을 포함하게 되므로, 기존 프롬프트 테스트는 깨지지 않음(추가 텍스트일 뿐, 기존 assert는 contains 기반).
