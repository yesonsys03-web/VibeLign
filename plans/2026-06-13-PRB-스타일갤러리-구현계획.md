# PR B — 스타일 갤러리 완성 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 디자인 프리뷰 갤러리를 5종으로 완성 — 나머지 4스타일(미니멀 SaaS·프루티거에어로·레트로다이너·리소그래프)에 토큰+레시피+모션을 베이크하고, 모든 목업 프롬프트에 공용 컴포넌트 커버리지 섹션을 추가한다.

**Architecture:** 4스타일은 `styles.ts`(TS SSOT)에 데이터로만 추가 — 각각 기존 `StyleSpec` 형태(tokens+recipe) + 모션 트랙에서 만든 optional `motion` 필드. Rust는 `StyleSpec`을 통째 역직렬화하므로 데이터 변경에 코드 수정 불필요. 공용 컴포넌트 커버리지는 `build_mockup_prompt`에 무조건 1개 섹션 추가(스타일·motion과 독립).

**Tech Stack:** React/TypeScript(Vitest), Rust(serde). 기반: 디자인 프리뷰 PR A + 모션 차원(완료).

**선행 의존:** 현재 `styles.ts`에는 `neo-brutalism` 1종(motion 포함)만 존재. `build_mockup_prompt`에는 `[규칙]` 블록 + (motion 있을 때) `[모션 — …]` 섹션이 있음.

**범위 밖(후속, design §6):** 각 스타일 정밀 육안 튜닝 반복 · 모션 강도 축 · 갤러리 모션 뱃지 · 자동 추천 · 내 디자인 스킬 저장.

```typescript
import { DESIGN_STYLES, getStyle } from "./styles";
// (기존 import에 이미 있으면 중복 추가 금지)

describe("DESIGN_STYLES 5종 완성", () => {
  it("5종이 순서대로 존재", () => {
    expect(DESIGN_STYLES.map((s) => s.id)).toEqual(
      ["neo-brutalism", "minimal-saas", "frutiger-aero", "retro-diner", "risograph"]);
  });
  it("모든 스타일이 motion(tokens+recipe)을 가진다", () => {
    for (const s of DESIGN_STYLES) {
      expect(s.motion, s.id).toBeDefined();
      expect(s.motion!.tokens.duration).toMatch(/ms|s/);
      expect(s.motion!.tokens.easing.length).toBeGreaterThan(0);
      expect(s.motion!.recipe.length).toBeGreaterThan(10);
    }
  });
  it("신규 4스타일은 토큰 9필드를 모두 가진다", () => {
    for (const id of ["minimal-saas", "frutiger-aero", "retro-diner", "risograph"]) {
      const t = getStyle(id)!.tokens;
      for (const k of ["bg","surface","text","primary","accent","border","fontFamily","radius","shadow"] as const) {
        expect(t[k], `${id}.${k}`).toBeTruthy();
      }
    }
  });
});
```
