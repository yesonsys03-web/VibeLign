## 1. 배경 / 문제

- VibeLign 프로그램을 **단일 상품**으로 파는 건 온라인·크라우드펀딩 판매에 약하다.
- 강 PD 제안: 프로그램 하나만 팔기보다 **학습 콘텐츠를 결합한 패키지**로 구성해야 펀딩에 유리하다.

## 2. 강 PD 제안 — 리워드 구성

### 2.1 다양한 구성품 결합 (번들)

프로그램(본체) + 부가 디지털 콘텐츠를 묶어 판매한다. 후보:

| 구성품 | 형태 | 역할 | 비고 |
|---|---|---|---|
| VibeLign 프로그램 | 앱(본체) | 핵심 상품 | 기존 |
| 전자책 | PDF | 개념·철학·사용 가이드 | 외부 매뉴얼과 연계(§2.2) |
| 강의 | VOD | 따라하기·실습 | 분량·범위 미정 |
| 자료집 | PDF/링크 | 용어 사전·예시 모음 | 초보자 가이드(§2.3)와 연계 |
| 템플릿 | 파일 | 바로 쓰는 기획/프로젝트 틀 | 종류 미정 |

→ **온라인 판매 유리**: 디지털 구성품은 한계비용 ~0, 티어 구성 자유로움.

### 2.2 매뉴얼의 외부화

- 강 PD: 프로그램 **내부 탭** 대신 **노션(Notion)/전자책** 형태로 매뉴얼 제공 → 사용자가 **작업 화면 옆에 띄워 놓고** 참고.
- **결정(2026-06-10): 둘 다 유지.**
  - 앱 내 **'사용법' 탭**(A안에서 우측 유틸로 이동, 커밋 9836462) — 빠른 진입용으로 존속.
  - **외부 상세 매뉴얼**(노션/전자책) — 옆에 띄워 참고 + **펀딩 번들의 구성품**으로 활용.
  - → 외부 매뉴얼이 곧 §2.1 번들의 한 구성품이 된다. 충돌이 아니라 역할 분리.

### 2.3 초보자용 가이드 강화

- **용어 사전(glossary)** — 코알못 대상.
- **단계별 하우투(How-to)** — "이걸 하려면 → 이 순서로".
- **무엇을 만들 수 있는가** 구체적 예시 — 사례/갤러리.
- → 이 콘텐츠들은 외부 매뉴얼(§2.2)·자료집(§2.1)의 내용물로 편입 가능.

## 3. 시너지 정리

§2.2 외부 매뉴얼 = §2.1 리워드 구성품 = §2.3 초보자 가이드의 그릇. **세 지적이 "외부 콘텐츠 자산" 하나로 수렴**한다 → 한 번 만들면 매뉴얼·번들·가이드 3역할.

## 4. 미결정 (비즈니스 의사결정 필요 — 임의로 정하지 않음)

- 리워드 **티어 구성·가격**
- VOD **분량·범위·제작 방식**
- 콘텐츠 **제작 일정·담당**
- 펀딩 **플랫폼**(와디즈/텀블벅 등)
- 번들 vs 단품 판매 정책

## 5. 다음 단계

1. 구성품별 **스코프·제작비** 산정 (전자책/VOD/자료집/템플릿)
2. **리워드 티어** 설계 (가격·구성 매트릭스)
3. (앱 측 소규모 연동) 외부 매뉴얼 **진입 링크**를 '사용법' 탭/온보딩에 추가 — §2.2 "둘 다 유지" 실현. 별도 작은 구현 작업.
