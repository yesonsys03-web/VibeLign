# 기획방 페르소나↔모델 분리 — 구현계획 4 (기획방 인라인 provider 편집)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 기획방 대화 화면(컴포저)에서 각 페르소나 칩에 붙은 작은 드롭다운으로, 설정 화면에 가지 않고도 그 역할이 쓸 모델(provider)을 즉시 바꾼다. 변경은 전역 설정에 저장되어 다음 호출부터 적용된다.

**Architecture:** 구현계획 2 의 lib 래퍼(`getPlanningPersonas`/`setPlanningPersonas`/`probePlanningProviders`)와 순수 로직(`effectivePersona`/`applyPersonaChange`/`PLANNING_PROVIDER_OPTIONS`, 이미 `PlanningPersonaSettings` 에서 export)을 재사용한다. 칩별 provider `<select>` 를 `PersonaProviderSelect` 컴포넌트로 분리하고, `PlanningPersonaComposer` 가 마운트 시 설정·설치목록을 읽어 각 칩 옆에 렌더한다. 기존 칩의 "이번 턴 포함" 토글 동작은 그대로 보존한다.

**Tech Stack:** React/TS, Vitest. 구현계획 1·2·3 완료 전제.

**관련:** 스펙 `...-design.md` §9.2, 구현계획 2(Settings UI·lib 래퍼).

## 범위 / 비범위

- **포함:** 컴포저의 각 페르소나 칩에 provider 드롭다운(claude/codex/agy/opencode, 미설치 주석). 변경 시 즉시 `setPlanningPersonas` 저장.
- **제외:** 칩에서의 ON/OFF. 컴포저 칩의 본래 토글은 "이번 턴에 이 페르소나 호출" 의미이고, 전역 enabled ON/OFF 는 개념이 달라 혼동을 부른다. 전역 ON/OFF 는 Settings(구현계획 2)에 유지.
- **제외:** 새 백엔드 변경 없음(구현계획 1·2 의 커맨드 그대로 사용).

## UX

- 칩은 `<button>`(토글) 단독이었으나, 이제 칩 컨테이너 `<div style={{display:inline-flex}}>` 안에 **[토글 버튼] + [provider select]** 를 나란히 둔다(버튼 안 버튼 금지 회피).
- select 변경 → 로컬 맵 갱신 + `setPlanningPersonas` 저장(낙관적). 실패해도 조용히 무시(.catch), 다음 로드시 동기화.
- 미설치 provider 는 옵션에 "(미설치)" 표기(선택은 허용 — 폴백이 받쳐줌).
- 설정/설치 로드 실패는 `.catch` 로 안전 처리(테스트 환경에서 invoke 미존재시 unhandled rejection 방지).

```tsx
// vibelign-gui/src/pages/planning/PersonaProviderSelect.tsx
import {
  PLANNING_PROVIDER_OPTIONS,
  effectivePersona,
} from "../../components/PlanningPersonaSettings";
import type { PlanningPersonaConfigMap } from "../../lib/vib";

export function providerOptionLabel(provider: string, installed: string[] | null): string {
  if (installed && !installed.includes(provider)) {
    return `${provider} (미설치)`;
  }
  return provider;
}

interface PersonaProviderSelectProps {
  readonly personaId: string;
  readonly map: PlanningPersonaConfigMap;
  readonly installed: string[] | null;
  readonly onChange: (provider: string) => void;
}

export function PersonaProviderSelect({ personaId, map, installed, onChange }: PersonaProviderSelectProps) {
  const eff = effectivePersona(map, personaId);
  return (
    <select
      aria-label={`${personaId} 모델`}
      value={eff.provider}
      onChange={(e) => onChange(e.target.value)}
      title="이 역할이 쓸 AI를 바꿔요. 없으면 다른 AI로 자동 대체돼요."
      style={{
        border: "2px solid #1A1A1A",
        background: "#FFFFFF",
        color: "#1A1A1A",
        fontSize: 10,
        fontWeight: 700,
        padding: "2px 4px",
        cursor: "pointer",
      }}
    >
      {PLANNING_PROVIDER_OPTIONS.map((p) => (
        <option key={p} value={p}>
          {providerOptionLabel(p, installed)}
        </option>
      ))}
    </select>
  );
}
```

- [ ] **Step 1: import + 상태 + 로드 추가.** 파일 상단 import 에 추가:
```tsx
import { getPlanningPersonas, setPlanningPersonas, probePlanningProviders, type PlanningPersonaConfigMap } from "../../lib/vib";
import { applyPersonaChange } from "../../components/PlanningPersonaSettings";
import { PersonaProviderSelect } from "./PersonaProviderSelect";

```
(파일은 이미 `import { useState } from "react";` 사용 중 — `useEffect` 를 같은 줄에 합치거나 별도 import. 기존 import 스타일에 맞춰 `import { useState, useEffect } from "react";` 로 통합 권장.)

컴포넌트 본문 상단(다른 useState 옆)에 추가:
```tsx
  const [personaMap, setPersonaMap] = useState<PlanningPersonaConfigMap>({});
  const [installedProviders, setInstalledProviders] = useState<string[] | null>(null);

  useEffect(() => {
    getPlanningPersonas().then(setPersonaMap).catch(() => setPersonaMap({}));
    probePlanningProviders().then(setInstalledProviders).catch(() => setInstalledProviders([]));
  }, []);

  function changeProvider(personaId: string, provider: string) {
    const next = applyPersonaChange(personaMap, personaId, { provider });
    setPersonaMap(next);
    void setPlanningPersonas(next).catch(() => {});
  }
```

- [ ] **Step 2: 칩 렌더를 컨테이너 + select 로 변경.** 기존 `{PLANNING_PERSONAS.map((persona) => ( <button ...>...</button> ))}` 블록(`:79-101`)을 다음으로 교체. **토글 `<button>` 자체(aria-pressed, onClick, 스타일)는 그대로 유지**하고 바깥을 `<div>` 로 감싸 select 를 붙인다:
```tsx
        {PLANNING_PERSONAS.map((persona) => (
          <div key={persona.id} style={{ display: "inline-flex", alignItems: "center", gap: 4 }}>
            <button
              type="button"
              aria-pressed={selectedPersonaIds.includes(persona.id)}
              onClick={() => {
                setSelectedPersonaIds(togglePersona(selectedPersonaIds, persona.id));
                setMessage((current) => appendMention(current, persona.mention));
              }}
              style={{
                border: "2px solid #1A1A1A",
                background: selectedPersonaIds.includes(persona.id) ? "#1A1A1A" : "#F7F0DF",
                color: selectedPersonaIds.includes(persona.id) ? "#FFFFFF" : "#1A1A1A",
                padding: "7px 9px",
                fontSize: 11,
                fontWeight: 900,
                cursor: "pointer",
              }}
            >
              <PlanningPersonaAvatar personaId={persona.id} label={persona.label} decorative size={18} />
              {persona.label} {persona.role}
            </button>
            <PersonaProviderSelect
              personaId={persona.id}
              map={personaMap}
              installed={installedProviders}
              onChange={(provider) => changeProvider(persona.id, provider)}
            />
          </div>
        ))}
```
(주의: `key` 가 이제 바깥 `<div>` 로 이동했으므로 `<button>` 의 `key` 는 제거. 나머지 버튼 속성/내용은 원본과 동일하게 유지.)
